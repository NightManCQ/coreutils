# 登录用户 (users)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**登录用户** 在汉语编程中对应英文命令：**users**。

显示当前登录系统的所有用户名，每出现一次表示一个登录会话。是最简单的用户查询工具。

**通俗理解**：
- **什么是登录用户**：就像点名一样，`登录用户` 命令会列出当前所有登录到系统的人。如果同一个人打开了多个终端窗口，他的名字会出现多次。
- **为什么需要它**：快速了解有多少人在使用系统，或者有哪些用户在线。比 `who` 命令更简洁，只显示用户名，没有其他信息。
- **应用场景**：快速检查、脚本编程、用户计数、系统监控、批量操作。

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `登录用户` |
| **拼音** | `dlyh` |
| **英文** | `users` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **帮助信息** | `登录用户 --帮助` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `登录用户 --版本` | `--version` | 显示命令的版本号并退出。 |

- `登录用户` 不接受任何其他参数。
- 它只显示用户名，用空格分隔。
- 同一用户的多个会话会显示多次。

## ✅ 执行结果介绍

### 基本执行
```bash
登录用户
```
**执行结果**：
```
用户 管理员 用户 开发者
```
**说明**：
- 显示所有登录用户的用户名
- 用空格分隔
- "用户" 出现两次，表示该用户有两个登录会话

### 单个用户
```bash
登录用户
```
**执行结果**：
```
用户
```
**说明**：只有一个用户登录。

### 无用户登录（极少见）
```bash
登录用户
```
**执行结果**：
```
（空输出）
```
**说明**：没有用户登录（通常在单用户模式或特殊情况下）。

### 失败情况
`登录用户` 几乎不会失败，除非 utmp 文件损坏：
```bash
登录用户
```
**可能的错误**（极少见）：
```
登录用户: 无法读取用户信息
```

⚠️ **注意**：
- 输出顺序可能与登录顺序不同。
- 只显示当前活跃的会话。
- 从 `/var/run/utmp` 文件读取信息。

## 🔍 底层逻辑详解

**登录用户** 的核心逻辑是 **"读取 utmp 文件中的用户名"**。

### 1. 信息来源

`登录用户` 从 `/var/run/utmp` 文件中读取信息：
- **utmp**：记录当前登录的用户
- **二进制格式**：需要专门解析
- **固定大小记录**：每条记录包含用户信息

### 2. users 的工作原理

当执行 `登录用户` 命令时：
1. **打开 utmp**：读取 `/var/run/utmp` 文件。
2. **遍历记录**：逐条检查 utmp 记录。
3. **过滤用户**：只保留类型为 USER_PROCESS 的记录。
4. **提取用户名**：从记录中提取用户名字段。
5. **输出结果**：将所有用户名用空格分隔输出。

### 3. utmp 结构

```c
struct utmp {
    short ut_type;              // 记录类型
    pid_t ut_pid;               // 进程ID
    char ut_line[UT_LINESIZE];  // 终端设备
    char ut_id[4];              // 初始化ID
    char ut_user[UT_NAMESIZE];  // 用户名 ← 这就是 users 输出的内容
    char ut_host[UT_HOSTSIZE];  // 远程主机
    struct exit_status ut_exit; // 退出状态
    long ut_session;            // 会话ID
    struct timeval ut_tv;       // 时间戳
};
```

### 4. 记录类型

`登录用户` 只关注以下类型：
- **USER_PROCESS (7)**：用户进程，表示活跃的用户会话

其他类型被忽略：
- EMPTY (0)、RUN_LVL (1)、BOOT_TIME (2) 等

### 5. 与相关命令的对比

| 命令 | 输出格式 | 详细信息 | 用途 |
|------|---------|---------|------|
| **users** | 用户名列表 | 最少 | 快速查看用户 |
| **who** | 详细列表 | 中等 | 用户+终端+时间 |
| **w** | 表格格式 | 最多 | 用户+活动+负载 |
| **last** | 历史记录 | 历史 | 登录历史 |
| **id** | 身份信息 | 单个用户 | UID/GID信息 |

### 6. 性能特点

#### 执行速度
- **非常快**：通常在毫秒级完成
- **简单解析**：只需读取和过滤 utmp 记录
- **无网络开销**：本地文件操作

#### 资源占用
- **内存**：极小（几KB）
- **CPU**：几乎为零
- **I/O**：读取一个小文件

### 7. 常见问题

#### 问题1：用户数与实际不符
**原因**：utmp 记录未及时清理。
**解决**：
```bash
# 检查 utmp 文件
sudo utmpdump /var/run/utmp

# 清理僵死记录（需要重启或特殊工具）
```

#### 问题2：看不到某些用户
**原因**：用户使用特殊登录方式或权限限制。
**解决**：使用 root 权限运行。

#### 问题3：输出顺序不一致
**原因**：utmp 记录的顺序可能变化。
**解决**：如需排序，使用 `sort` 命令。

### 8. 安全性考虑

#### 信息泄露
- users 显示所有登录用户
- 可能被用于社会工程攻击
- 应限制对 utmp 文件的访问

#### 防护措施
```bash
# 检查 utmp 权限
ls -la /var/run/utmp
# 应该是 -rw-rw-r-- root utmp

# 确保只有授权用户可以读取
```

### 9. 最佳实践

#### 脚本中使用
```bash
# 推荐：结合其他命令使用
用户数=$(登录用户 | wc -w)
唯一用户=$(登录用户 | tr ' ' '\n' | sort -u)

# 避免：依赖输出顺序
第一个用户=$(登录用户 | awk '{print $1}')  # 不可靠
```

#### 监控脚本
```bash
# 定期检查并记录
*/5 * * * * /usr/bin/登录用户 >> /var/log/用户.log
```

#### 告警阈值
```bash
# 设置合理的阈值
如果 [ $(登录用户 | wc -w) -gt 20 ]; then
    发送告警
结束
```

## 📊 使用频率说明

在现代 Linux 系统中，`登录用户` (users) 的使用频率**低到中等**。

*   **原因**：
    *   功能过于简单，通常使用 `who` 或 `w`。
    *   在脚本中用于快速计数。
    *   个人桌面系统很少使用。

*   **替代方案**：
    *   **who**：提供更多详细信息
    *   **w**：显示用户活动
    *   **who -q**：类似 users 但带计数
    *   **图形界面**：系统监视器

*   **推荐使用场景**：
    *   **快速检查**：最简单的方式查看有谁在线
    *   **脚本编程**：用户计数和列表
    *   **批量操作**：向所有用户发送消息
    *   **监控告警**：用户数量监控
    *   **教学演示**：讲解 utmp 机制

## 📜 Linux 中的定义与发展史

### 1. Unix 早期的用户管理
1970年代的Unix就包含了 users 命令。它是最简单的用户查询工具，用于快速查看谁在使用系统。

### 2. utmp 机制的确立
Unix早期就建立了utmp机制来跟踪用户登录，users 命令直接从这个文件读取信息。

### 3. POSIX 标准化
POSIX 标准对 users 的行为进行了规范，定义了基本的输出格式。

### 4. GNU Coreutils 的实现
GNU项目将 users 整合到GNU Coreutils包中，保持了简洁的设计哲学。

### 5. 现代登录方式的变化
随着SSH、图形界面等新的登录方式出现，users 仍然有效，因为它基于底层的utmp机制。

### 6. 容器化时代的挑战
在容器环境中，users 可能不适用：
- 容器通常共享主机的 utmp
- 或者有独立的 utmp 文件
- 需要特别注意上下文

### 7. uutils 的 Rust 重写
uutils项目用Rust重写了 users，提供了内存安全保证和跨平台兼容性。

### 8. 现代应用生态
如今，`登录用户` 被应用于：
- **脚本编程**：简单的用户查询
- **系统监控**：用户数量统计
- **安全管理**：在线用户检查
- **自动化运维**：批量操作用户

### 9. 未来的地位
虽然 users 功能简单，但其简洁性使其在脚本编程中仍有价值。未来，它将继续作为最快速的用戶查询工具存在。
