# 创建文夹 (mkdir)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**创建文夹** 在汉语编程中对应英文命令：**mkdir**。

在文件系统中开辟一个新的“房间”（文件夹），用于组织和存放文件。

**通俗理解**：
就像是在大楼里新建一个房间。你可以只建一个房间，也可以一次性建一整层楼（多级目录）。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `创建文夹 新建文件夹` |
| **英文** | `mkdir new_folder` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **递归创建** | `创建文夹 递归 项目/src/utils` | `-p` / `--parents` | 自动创建所有缺失的父目录，如果目录已存在也不报错。 |
| **设置权限** | `创建文夹 权限 755 共享文件夹` | `-m [模式]` / `--mode=[模式]` | 在创建时直接设置文件夹的权限（如755、700）。 |
| **详细输出** | `创建文夹 详细 新目录` | `-v` / `--verbose` | 显示每个被创建的目录名称。 |
| **安全上下文** | `创建文夹 上下文 unconfined_u:object_r:user_home_t:s0 文件夹` | `-Z` / `--context` | 设置 SELinux 安全上下文（仅支持 SELinux 系统）。 |
| **帮助信息** | `创建文夹 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `创建文夹 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

在指定位置建立一个空的文件夹。如果使用 `-p` 参数，即使父目录不存在也会自动创建。

## 🔍 底层逻辑详解

**创建文夹** 命令的本质是在文件系统的目录结构中增加一个新的条目。

### 1. 目录的本质
- **文件夹是什么**：在 Linux 中，文件夹其实也是一种特殊的“文件”，它的内容是一张清单，记录了里面有哪些文件和子文件夹。
- **创建过程**：系统在磁盘上分配一块空间，写入这个“清单文件”，并在父目录的清单里加上这个新房间的名字。

### 2. 递归创建（-p）的魔力
- **普通模式**：如果你想建 `A/B/C`，但 `A` 和 `B` 都不存在，系统会报错：“找不到路径”。
- **递归模式**：系统会像搭积木一样，先检查 `A` 在不在，不在就建 `A`；再检查 `B`，不在就建 `B`；最后建 `C`。一步到位，非常省心。

### 3. 权限与所有权
- **默认权限**：新建的文件夹通常会继承系统的默认权限（受 `umask` 影响），一般是 `755` 或 `775`。
- **自定义权限**：使用 `-m` 参数可以在创建的一瞬间就定好规矩，比如设为 `700` 只有你自己能进。

### 4. 为什么需要这个命令？
- **组织数据**：就像整理房间需要不同的抽屉和柜子，文件系统需要文件夹来分类存放数据。
- **程序需求**：很多软件在运行前需要先准备好特定的目录结构（如日志目录、配置目录）。
