# 打印格式 (pr)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**打印格式** 在汉语编程中对应英文命令：**pr**。

将文本文件转换为适合打印的格式，自动添加页眉、页脚、页码和分页符。

**通俗理解**：
- **排版助手**：就像是你写了一篇长文章想打印出来，但直接打印会显得乱糟糟的。`打印格式` 命令就像一个专业的排版员，它会帮你加上页码、日期、标题，并把内容整齐地分成一页一页的。
- **分页大师**：它能确保你的文字不会在打印时被切断，每一页都有清晰的边界和标识。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `打印格式 报告.txt` |
| **拼音** | `dygs` |
| **英文** | `pr report.txt` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **列数** | `打印格式 --列数 2 文本.txt` | `-2` / `--columns=2` | 将输出分为多列显示，默认为 1 列。 |
| **页长** | `打印格式 --页长 50 文本.txt` | `-l [数字]` / `--length=[数字]` | 设置每页的行数，默认为 66 行。 |
| **页宽** | `打印格式 --页宽 72 文本.txt` | `-w [数字]` / `--width=[数字]` | 设置每页的宽度（字符数）。 |
| **页眉标题** | `打印格式 --页眉 "季度总结" 报告.txt` | `-h [标题]` / `--header=[标题]` | 自定义每页顶部的标题文字。 |
| **省略头尾** | `打印格式 --省略头尾 文本.txt` | `-t` / `--omit-header` | 不打印页眉和页脚，只输出正文。 |
| **帮助信息** | `打印格式 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `打印格式 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

默认情况下，输出内容会被分成每页 66 行，每页顶部有包含文件名、日期和页码的页眉，底部有分页符。

## 💻 详细用法举例

### 1. 基础用法 - 准备打印文档
```bash
打印格式 年度报告.txt
```
**执行结果**：屏幕会显示格式化后的内容，每页顶部有类似 `2024-04-11 10:30    年度报告.txt    Page 1` 的页眉。

### 2. 双栏排版
```bash
打印格式 --列数 2 代码清单.txt
```
**执行结果**：将文本分成左右两栏显示，这在打印代码或对照表时非常节省纸张。

### 3. 自定义页眉
```bash
打印格式 --页眉 "机密文档" --页长 50 内部备忘录.txt
```
**执行结果**：每页顶部都会显示"机密文档"作为标题，并且每页只有 50 行，适合小型打印机。

### 4. 纯净输出（无页眉页脚）
```bash
打印格式 --省略头尾 数据.txt > 干净数据.txt
```
**执行结果**：去掉了所有的页眉页脚和分页符，只保留原始文本内容。这在预处理数据时很有用。

## 🔍 底层逻辑详解

**打印格式** (pr) 的核心逻辑是 **"分页计算"** 与 **"页面装饰"**。

### 1. 它是如何工作的？
- **计数分行**：`打印格式` 会逐行读取输入，并计算当前是第几行。
- **触发分页**：当行数达到设定的页长（默认 66 行）时，它会插入一个分页符（Form Feed），并开始新的一页。
- **添加装饰**：在每一页的顶部，它会生成包含日期、文件名和页码的页眉；在底部留白作为页脚区域。

### 2. 多栏排版的实现
*   **虚拟网格**：当使用 `-2` 或更多列时，`打印格式` 会在内存中维护一个虚拟的页面网格。
*   **填充策略**：它先填满第一栏，再填第二栏。这种"先竖后横"的填充方式确保了阅读的逻辑性。
*   **宽度均分**：它会自动计算每栏的宽度，确保总宽度不超过设定的页宽。

### 3. 与 fmt 的区别
*   **pr (打印格式)**：关注的是"页面"的概念。它有明确的页眉、页脚和分页符，是为了物理打印而设计的。
*   **fmt (格式化)**：关注的是"段落"的概念。它只是简单地调整行宽，不关心分页，更适合屏幕阅读。

### 4. 性能表现
`打印格式` 是一个流式处理工具，但在处理多栏排版时，它需要一定的缓冲区来暂存数据，以便进行栏间对齐。因此，在处理极大的文件时，它的内存占用会比简单的 `cat` 稍高一些。

## 📊 使用频率说明

在现代 Linux 系统中，`打印格式` (pr) 的使用频率**较低**。
*   **原因**：随着图形化界面和现代办公软件（如 LibreOffice、Word）的普及，人们很少再通过命令行来排版打印文档。
*   **替代方案**：推荐使用 `pandoc` 进行文档转换，或使用 `enscript`、`a2ps` 等更专业的打印工具。
*   **特殊场景**：在自动化脚本中生成纯文本报表、在嵌入式系统中快速预览文档，或者需要将多个文件合并成一个带页码的打印任务时，`pr` 依然非常高效。

## 📜 Linux 中的定义与发展史

### 1. 电传打字机时代的产物
`pr` 的名字来源于 "Print"（打印）。在早期的计算机时代，输出设备主要是电传打字机和行式打印机。为了让打印出来的纸张整洁美观，需要一个工具来自动处理分页和页眉，`pr` 因此诞生。

### 2. Unix V4 的引入
`pr` 命令最早出现在 1973 年的 Unix V4 中。它是当时 Unix 办公自动化套件的重要组成部分，常与 `lp` (打印守护进程) 配合使用。

### 3. POSIX 标准的固化
POSIX 标准规定了 `pr` 必须支持多栏排版、自定义页长和页眉。这使得它在不同 Unix-like 系统上的行为保持高度一致。

### 4. 现代系统中的角色
虽然现在很少有人直接用 `pr` 来驱动打印机，但它在以下领域依然有一席之地：
*   **日志归档**：将超长的日志文件分割成带页码的片段，方便人工查阅。
*   **代码审查**：在终端里以双栏形式并排显示两个版本的代码，进行快速对比。
*   **报表生成**：在 Shell 脚本中自动生成带有标题和日期的纯文本统计报表。
