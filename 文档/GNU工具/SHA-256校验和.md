# SHA-256校验和 (sha256sum)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**SHA-256校验和** 在汉语编程中对应英文命令：**sha256sum**。

计算并验证文件的 SHA-256（Secure Hash Algorithm 256-bit）消息摘要。用于检查文件完整性、数字签名、密码学应用等安全敏感场景。

**通俗理解**：
- **什么是SHA-256校验和**：就像给文件生成一个更安全的"数字指纹"，`SHA-256校验和` 命令会读取文件内容并生成一个64位的十六进制字符串（256位）。与MD5相比，SHA-256更安全，目前还没有已知的实用碰撞攻击方法。它广泛用于需要高安全性的场景，如软件签名、区块链、证书验证等。
- **为什么需要它**：当你需要确保文件不仅完整，而且没有被恶意篡改时，SHA-256比MD5更可靠。特别是在软件分发、安全审计、区块链等场景中，SHA-256是行业标准。
- **应用场景**：软件包签名验证、区块链、SSL证书、密码存储、Git版本控制、安全审计、数据完整性保证。

### SHA-256 特性

- **固定长度**：始终生成64位十六进制字符串（256位）
- **单向性**：无法从哈希值反推原始内容
- **抗碰撞**：目前无已知实用碰撞攻击
- **雪崩效应**：输入微小变化导致输出巨大变化
- **安全性**：被NIST推荐，广泛用于安全领域

## 🔍 底层逻辑详解

**SHA-256校验和** 的核心逻辑是 **"实现SHA-256哈希算法"**。

### 1. SHA-256 算法概述

SHA-256 是 SHA-2 家族的一员，由 NSA 设计，NIST 发布。

#### 主要特点
- **输出长度**：256位（32字节）
- **块大小**：512位（64字节）
- **轮数**：64轮
- **初始值**：8个32位常量（源自素数的平方根）

### 2. 算法步骤

#### 步骤1：预处理
```
1. 附加比特 '1'
2. 填充 k 个 '0'，使长度 ≡ 448 (mod 512)
3. 附加64位原始消息长度
```

#### 步骤2：初始化哈希值
```
H0 = 0x6a09e667
H1 = 0xbb67ae85
H2 = 0x3c6ef372
H3 = 0xa54ff53a
H4 = 0x510e527f
H5 = 0x9b05688c
H6 = 0x1f83d9ab
H7 = 0x5be0cd19
```

#### 步骤3：处理每个512位块
```
对于每个块:
    准备消息调度 W[0..63]
    
    初始化工作变量 a,b,c,d,e,f,g,h
    
    64轮压缩:
        T1 = h + Σ1(e) + Ch(e,f,g) + K[t] + W[t]
        T2 = Σ0(a) + Maj(a,b,c)
        
        h = g
        g = f
        f = e
        e = d + T1
        d = c
        c = b
        b = a
        a = T1 + T2
    
    更新哈希值:
        H0 += a, H1 += b, ..., H7 += h
```

#### 步骤4：输出
```
SHA-256 = H0 || H1 || H2 || H3 || H4 || H5 || H6 || H7
```

### 3. 与 MD5 的对比

| 特性 | MD5 | SHA-256 |
|------|-----|---------|
| **输出长度** | 128位 | 256位 |
| **块大小** | 512位 | 512位 |
| **轮数** | 64轮（4轮×16） | 64轮 |
| **安全性** | 已破解 | 安全 |
| **速度** | 快 | 较慢 |
| **推荐用途** | 完整性检查 | 安全应用 |

### 4. 系统调用层面

```c
#include <stdio.h>
#include <openssl/sha.h>

unsigned char hash[SHA256_DIGEST_LENGTH];
SHA256_CTX ctx;

SHA256_Init(&ctx);

FILE *fp = fopen(filename, "rb");
char buffer[4096];
size_t bytes;

while ((bytes = fread(buffer, 1, sizeof(buffer), fp)) > 0) {
    SHA256_Update(&ctx, buffer, bytes);
}

SHA256_Final(hash, &ctx);
fclose(fp);

// 输出十六进制
for (int i = 0; i < SHA256_DIGEST_LENGTH; i++) {
    printf("%02x", hash[i]);
}
```

### 5. 安全性分析

#### 抗碰撞性
- **理论强度**：2^128 次操作才能找到碰撞
- **现状**：无已知实用攻击
- **推荐**：NIST推荐用于安全应用

#### 抗第二原像
- **强度**：2^256 次操作
- **现状**：安全

#### 抗原像
- **强度**：2^256 次操作
- **现状**：安全

## 📜 Linux 中的定义与发展史

### 1. SHA 家族的发展
- **1993年**：SHA-0 发布（很快发现缺陷）
- **1995年**：SHA-1 发布
- **2001年**：SHA-2 家族发布（包括SHA-256）
- **2015年**：SHA-3 发布

### 2. SHA-256 的标准化
2001年，NIST将SHA-256作为FIPS 180-2标准发布。

### 3. 广泛应用
- **比特币**：2009年采用SHA-256作为工作量证明
- **TLS/SSL**：证书签名
- **Git**：对象哈希（虽然用SHA-1，但计划迁移到SHA-256）

### 4. GNU Coreutils 的实现
GNU项目提供了sha256sum命令。

### 5. uutils 的 Rust 重写
uutils项目用Rust重写了sha256sum。

### 6. 现代应用生态
如今，`SHA-256校验和` 被应用于：
- **加密货币**：比特币、以太坊等
- **区块链**：各种区块链平台
- **软件分发**：Linux发行版、软件包
- **版本控制**：Git（计划迁移）
- **容器技术**：Docker镜像摘要
- **网络安全**：TLS证书、代码签名

### 7. 云原生时代
在云环境中，SHA-256无处不在：
- **容器registry**：镜像摘要
- **对象存储**：数据完整性
- **CI/CD**：构建产物验证
- **密钥管理**：密钥指纹

### 8. 未来的地位
SHA-256将在很长一段时间内继续作为标准：
- **标准化**：NIST标准
- **普及性**：行业广泛采用
- **安全性**：目前仍然安全
- **兼容性**：大量现有系统依赖

未来可能会出现向SHA-3或BLAKE2的迁移，但SHA-256由于其普及性和足够的安全性，将长期存在。
