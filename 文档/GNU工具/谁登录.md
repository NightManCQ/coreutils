# 谁登录 (who)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**谁登录** 在汉语编程中对应英文命令：**who**。

显示当前登录系统的用户信息，包括用户名、终端、登录时间和来源主机。是监控系统用户活动的重要工具。

**通俗理解**：
- **什么是谁登录**：就像查看"现在谁在家里"，`谁登录` 命令会告诉你当前有哪些用户登录了系统，他们从哪个终端登录的，什么时候登录的，以及从哪里（本地或远程）登录的。
- **为什么需要它**：在多用户系统中，了解谁在使用系统非常重要。系统管理员需要监控用户活动，确保系统安全，诊断问题，或者只是想知道是否有其他人在使用系统。
- **应用场景**：用户监控、安全管理、故障排查、审计追踪、系统维护。

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `谁登录` |
| **拼音** | `sdl` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **所有信息** | `谁登录 --所有` | `-a` / `--all` | 显示所有可用信息，等价于 `-b -d --login -p -r -t -T -u`。 |
| **启动时间** | `谁登录 --启动` | `-b` / `--boot` | 显示系统最后一次启动的时间。 |
| **死亡进程** | `谁登录 --死亡` | `-d` / `--dead` | 显示已终止的进程。 |
| **头部标题** | `谁登录 --头部` | `-H` / `--heading` | 显示列标题行。 |
| **登录进程** | `谁登录 --登录` | `--login` | 显示系统登录进程。 |
| **仅主机** | `谁登录 --仅主机` | `-l` / `--lookup` | 尝试通过 DNS 查找主机名。 |
| **消息状态** | `谁登录 --消息` | `-T` / `-w` / `--mesg` | 显示用户的消息状态（+ 可写，- 不可写，? 未知）。 |
| **进程ID** | `谁登录 --进程` | `-p` / `--process` | 显示由 init 产生的进程。 |
| **运行级别** | `谁登录 --运行级别` | `-r` / `--runlevel` | 显示当前运行级别。 |
| **空闲时间** | `谁登录 --空闲` | `-u` / `--users` | 显示用户的空闲时间。 |
| **简短输出** | `谁登录 --简短` | `-q` / `--count` | 只显示用户名和登录用户数。 |
| **帮助信息** | `谁登录 --帮助` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `谁登录 --版本` | `--version` | 显示命令的版本号并退出。 |

### 输出格式说明

#### 默认输出
```bash
谁登录
```
**输出**：
```

管理员   pts/1        2024-01-15 09:15 (:0)
```
**说明**：依次显示：用户名、终端、登录日期时间、来源主机。

#### 简短输出
```bash
谁登录 --简短
```
**输出**：
```
用户 管理员
# 用户数=2
```
**说明**：只显示用户名和用户数量。

```bash

```


**说明**：
- **用户**：登录的用户名
- **pts/0**：终端设备（伪终端）
- **2024-01-15 10:30**：登录时间
- **(192.168.1.100)**：来源IP地址（SSH登录）或显示 `(:0)`（本地图形界面）

```bash

```
**执行结果**：
```

运行级别 5         2024-01-10 07:10
登录      tty1     2024-01-15 08:00              1234 id=tty1
用户     + pts/0   2024-01-15 10:30 00:15       5678 (192.168.1.100)
管理员   + pts/1   2024-01-15 09:15  .          9012 (:0)
```
**说明**：显示更详细的信息，包括消息状态、空闲时间、进程ID等。

### 简短输出
```bash
谁登录 --简短
```
**执行结果**：
```
用户 管理员
# 用户数=2
```
**说明**：简洁的用户列表和计数。

### 失败情况
如果没有用户登录（极少见），可能只显示标题或为空：
```bash
谁登录
```
**可能的输出**：
```
（空输出）
```

⚠️ **注意**：
- 只能看到自己和其他普通用户的登录信息。
- root 用户可以看到所有用户的详细信息。
- 某些字段可能在某些系统上不可用。

## 🔍 底层逻辑详解

**谁登录** 的核心逻辑是 **"读取 utmp 文件中的登录信息"**。

### 1. 信息来源

`谁登录` 主要从以下文件获取信息：

#### /var/run/utmp
- 记录当前登录的用户
- 二进制格式
- 包含字段：用户名、终端、登录时间、主机等

#### /var/log/wtmp
- 记录所有登录和注销历史
- 用于 `last` 命令
- 二进制格式

#### /var/log/btmp
- 记录失败的登录尝试
- 用于 `lastb` 命令

### 2. who 的工作原理

当执行 `谁登录` 命令时：
1. **读取 utmp**：打开 `/var/run/utmp` 文件。
2. **解析记录**：逐条读取 utmp 记录。
3. **过滤有效会话**：只保留有效的用户会话。
4. **格式化输出**：根据参数格式化输出。
5. **显示结果**：打印到标准输出。

### 3. utmp 结构

```c
struct utmp {
    short ut_type;              // 记录类型
    pid_t ut_pid;               // 进程ID
    char ut_line[UT_LINESIZE];  // 终端设备
    char ut_id[4];              // 初始化ID
    char ut_user[UT_NAMESIZE];  // 用户名
    char ut_host[UT_HOSTSIZE];  // 远程主机
    struct exit_status ut_exit; // 进程退出状态
    long ut_session;            // 会话ID
    struct timeval ut_tv;       // 时间戳
    int32_t ut_addr_v6[4];      // IP地址
};
```

### 4. 记录类型

| 类型 | 值 | 说明 |
|------|-----|------|
| EMPTY | 0 | 空记录 |
| RUN_LVL | 1 | 运行级别变化 |
| BOOT_TIME | 2 | 系统启动 |
| NEW_TIME | 3 | 时间改变 |
| OLD_TIME | 4 | 旧时间 |
| INIT_PROCESS | 5 | init 进程 |
| LOGIN_PROCESS | 6 | 登录进程 |
| USER_PROCESS | 7 | 用户进程 |
| DEAD_PROCESS | 8 | 已终止进程 |

### 5. 与相关命令的对比

| 命令 | 信息来源 | 详细程度 | 用途 |
|------|---------|---------|------|
| **who** | utmp | 中等 | 当前登录用户 |
| **w** | utmp + proc | 详细 | 用户及其活动 |
| **users** | utmp | 简单 | 仅用户名列表 |
| **last** | wtmp | 历史 | 登录历史 |
| **lastlog** | lastlog | 历史 | 最后登录时间 |

### 6. 消息状态

#### 消息状态符号
- **+**：允许接收消息（write 命令）
- **-**：禁止接收消息
- **?**：无法确定

#### 控制消息接收
```bash
# 允许接收消息
mesg y

# 禁止接收消息
mesg n

# 检查状态
mesg
```

### 7. 终端类型

#### 常见终端
- **ttyN**：虚拟控制台（Ctrl+Alt+F1-F6）
- **pts/N**：伪终端（SSH、图形终端）
- **:0**：本地图形界面（X11）

#### 终端设备文件
```bash
ls -la /dev/tty*
ls -la /dev/pts/*
```

### 8. 安全性考虑

#### 隐私问题
- who 显示用户的登录信息
- 包括来源IP地址
- 可能被用于社会工程攻击

#### 防护措施
- 限制 utmp 文件的读取权限
- 使用 fail2ban 防止暴力破解
- 监控异常登录模式

### 9. 常见问题

#### 问题1：显示的用户与实际不符
**原因**：utmp 记录未正确清理。
**解决**：
```bash
# 手动清理
sudo rm /var/run/utmp
sudo touch /var/run/utmp
```

#### 问题2：看不到某些用户
**原因**：权限限制或用户使用特殊登录方式。
**解决**：使用 root 权限运行。

#### 问题3：主机名显示为IP
**原因**：DNS 解析失败或未启用。
**解决**：
```bash
谁登录 --仅主机
```

## 📊 使用频率说明

在现代 Linux 系统中，`谁登录` (who) 的使用频率**中等**。

*   **原因**：
    *   系统管理员经常检查登录用户。
    *   多用户系统中用于监控用户活动。
    *   个人桌面系统使用较少。

*   **替代方案**：
    *   **w**：更详细的用户信息
    *   **users**：更简洁的用户列表
    *   **last**：登录历史记录
    *   **图形界面**：系统监视器

*   **推荐使用场景**：
    *   **用户监控**：查看谁在使用系统
    *   **安全管理**：检测异常登录
    *   **技术支持**：了解系统使用情况
    *   **审计追踪**：记录用户活动
    *   **资源管理**：评估系统负载

## 📜 Linux 中的定义与发展史

### 1. Unix 早期的用户管理
1970年代的Unix就包含了 who 命令。它是最早的用户管理工具之一，用于查看谁在使用系统。

### 2. utmp/wtmp 机制的确立
Unix早期就建立了utmp/wtmp机制来跟踪用户登录，这个机制一直沿用至今。

### 3. POSIX 标准化
POSIX 标准对 who 的行为进行了规范，定义了输出格式和基本选项。

### 4. GNU Coreutils 的实现
GNU项目将 who 整合到GNU Coreutils包中，提供了丰富的选项和更好的兼容性。

### 5. 网络时代的影响
随着网络的发展，who 增加了显示远程主机信息的功能，支持SSH等远程登录方式。

### 6. 现代登录方式的变化
- **SSH**：加密的远程登录
- **图形界面**：GDM、KDM等显示管理器
- **单点登录**：集中认证系统

### 7. 容器化时代的挑战
在容器环境中，传统的 who 命令可能不适用：
- 容器通常没有传统登录会话
- 用户通过 exec 进入容器
- 需要新的监控方式

### 8. uutils 的 Rust 重写
uutils项目用Rust重写了 who，提供了内存安全保证和跨平台兼容性。

### 9. 现代应用生态
如今，`谁登录` 被应用于：
- **系统管理**：用户活动监控
- **安全审计**：登录记录追踪
- **合规性**：满足审计要求
- **资源管理**：评估系统使用情况

### 10. 未来的地位
随着身份管理技术的发展，传统 who 的概念可能在某些场景中逐渐淡化，但在多用户系统和服务器管理中，它仍然是重要的监控工具。
