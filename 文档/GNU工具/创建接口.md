# 创建接口 (mknod)

> **对联系统理念**: 以日常汉语为导向，精准表意，拒绝冗余。

## 📋 核心定义

**创建接口** 在汉语编程中对应英文命令：**mknod**。

给硬件设备或通信管道安装一个“插座”（接口文件），让程序可以像插拔电器一样进行数据交换或硬件控制。

**通俗理解**：
- **拒绝重复**：Linux 里有个叫 `mkfifo` 的命令专门用来建通信管道，其实它就是 `mknod p` 的“马甲”。我们填平这个历史坑，统一用 **创建接口** 来安装所有类型的“插座”。
- **插座逻辑**：无论是连硬盘（块设备）、连键盘（字符型）还是程序间对话（发布件/管道），本质上都是在系统墙上安一个插座，插上就能通电（通数据）。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `创建接口 虚拟设备 字符型 1 3` |
| **英文** | `mknod /dev/virtual_dev c 1 3` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **类型：字符型** | `创建接口 设备名 字符型 主号 次号` | `c` / `--type=c` | 创建字符设备接口，数据像水流一样逐个字节传输（如键盘、鼠标）。 |
| **类型：块设备** | `创建接口 磁盘镜像 块设备 8 0` | `b` / `--type=b` | 创建块设备接口，数据按块整体读写（如硬盘、U盘）。 |
| **类型：发布件** | `创建接口 通信口 发布件 0 0` | `p` / `--type=p` | 创建一个用于进程间通信的“数据插座”（即命名管道）。 |
| **设置权限** | `创建接口 权限 666 共享设备` | `-m [模式]` / `--mode=[模式]` | 在创建时直接设置接口的访问权限。 |
| **帮助信息** | `创建接口 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `创建接口 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

在文件系统中生成一个特殊的设备文件。它看起来像个普通文件，但实际上是通往硬件内核驱动或通信管道的“传送门”。

## 🔍 底层逻辑详解

**创建接口** (mknod) 是 Linux 系统管理中最硬核的命令之一，它直接关联到操作系统的核心——设备驱动。

### 1. 为什么需要“插座”？
- **统一入口**：Linux 的哲学是“一切皆文件”。为了让程序能统一地控制各种奇形怪状的硬件，系统给每个硬件都配了一个文件形式的“插座”。
- **身份标识**：每个插座都有两个编号：**主设备号**（决定是哪个驱动程序）和**次设备号**（决定是该驱动下的哪个具体设备）。

### 2. 两种主要的“插头”类型
- **字符型 (c)**：像水管里的水，数据是一个字节一个字节连续流动的。适合键盘、鼠标等。
- **块设备 (b)**：像集装箱，数据是一整块一整块搬运的。适合硬盘、光盘等。
- **发布件 (p)**：这是一个“内部对讲机”插座。它不存数据，只负责把一端的数据实时引到另一端（实现发布-订阅模型）。

### 3. 翻译灾难：关于“套接字” (Socket)
在技术文档中，`p` 类型常被误称为“本地套接字”。这是一个典型的**翻译灾难**：
- **Socket 的本意**：就是墙上的**“插座”**。
- **我们的立场**：我们拒绝使用“套接字”这种制造壁垒的词。在 `mknod` 中，它就是 **“发布件”** 插座；在网络中，它就是 **“网络连线口”**。

### 4. 现代系统中的角色
在现代 Linux 中，大部分设备接口都是由系统自动创建的。但在嵌入式开发、容器技术或调试底层驱动时，手动使用 `创建接口` 依然是必备技能。
