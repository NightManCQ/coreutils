# 终端设置 (stty)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**终端设置** 在汉语编程中对应英文命令：**stty**。

显示和修改终端线路设置。用于配置终端的行为，如回显、特殊字符、波特率、行 discipline 等。

**通俗理解**：
- **什么是终端设置**：就像调节电视机的各种参数（亮度、对比度、音量），`终端设置` 命令可以调整终端的各种行为。比如是否显示你输入的字符（回显）、按 Ctrl+C 时做什么、终端的速度等。这些设置控制着你与计算机交互的方式。
- **为什么需要它**：有时候终端的行为不符合你的期望，比如输入密码时仍然显示字符、退格键不起作用、特殊按键没有预期效果等。使用 `终端设置` 可以调整这些行为，让终端按照你想要的方式工作。
- **应用场景**：密码输入隐藏、终端故障修复、脚本中的终端控制、嵌入式设备配置、串口通信。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `终端设置` |
| **拼音** | `z端sz` |

### 参数详解

#### 查询设置

| 参数 | 示例 | 作用说明 |
|------|------|---------|
| **显示所有** | `终端设置 --all` | `-a` / `--all` 显示所有终端设置。 |
| **保存设置** | `终端设置 --save` | `-g` / `--save` 以可恢复的格式显示当前设置。 |
| **帮助信息** | `终端设置 --帮助` | `--help` 显示完整的帮助文档。 |
| **版本信息** | `终端设置 --版本` | `--version` 显示版本号并退出。 |

#### 常用设置

| 设置 | 开启 | 关闭 | 作用说明 |
|------|------|------|---------|
| **回显** | `echo` | `-echo` | 控制是否显示输入的字符。 |
| **规范模式** | `icanon` | `-icanon` | 启用/禁用行缓冲和特殊字符处理。 |
| **Ctrl+C中断** | `isig` | `-isig` | 启用/禁用信号生成（Ctrl+C、Ctrl+Z等）。 |
| **回车换行** | `icrnl` | `-icrnl` | 将回车符转换为换行符。 |
| **小写转大写** | `iuclc` | `-iuclc` | 将小写字母转换为大写。 |
| **流控制** | `ixon` | `-ixon` | 启用/禁用 XON/XOFF 流控制（Ctrl+S/Ctrl+Q）。 |

#### 特殊字符设置

| 字符 | 设置 | 默认值 | 作用说明 |
|------|------|--------|---------|
| **中断** | `intr` | Ctrl+C | 发送 SIGINT 信号。 |
| **退出** | `quit` | Ctrl+\ | 发送 SIGQUIT 信号。 |
| **擦除** | `erase` | Backspace | 删除前一个字符。 |
| **杀死行** | `kill` | Ctrl+U | 删除整行。 |
| **文件结束** | `eof` | Ctrl+D | 表示输入结束。 |
| **挂起** | `susp` | Ctrl+Z | 发送 SIGTSTP 信号。 |
| **开始输出** | `start` | Ctrl+Q | 恢复输出（XON）。 |
| **停止输出** | `stop` | Ctrl+S | 暂停输出（XOFF）。 |

#### 波特率设置（串口）

| 波特率 | 命令 | 说明 |
|--------|------|------|
| **9600** | `终端设置 9600` | 标准串口速度。 |
| **115200** | `终端设置 115200` | 高速串口。 |
| **其他** | `终端设置 速度` | 支持多种波特率。 |

## 🔍 底层逻辑详解

**终端设置** 的核心逻辑是 **"通过 ioctl 系统调用配置终端"**。

### 1. 工作原理

当执行 `终端设置` 命令时：
1. **打开终端**：获取标准输入的文件描述符。
2. **查询当前设置**：调用 `tcgetattr()` 获取当前 termios 结构。
3. **修改设置**：根据命令行参数修改 termios 结构。
4. **应用设置**：调用 `tcsetattr()` 应用新设置。
5. **返回**：成功返回 0，失败返回错误码。

### 2. 系统调用层面

#### tcgetattr() 和 tcsetattr()
```c
#include <termios.h>

struct termios tty;

// 获取当前设置
tcgetattr(STDIN_FILENO, &tty);

// 修改设置
tty.c_lflag &= ~ECHO;  // 禁用回显

// 应用设置
tcsetattr(STDIN_FILENO, TCSANOW, &tty);
```

#### termios 结构
```c
struct termios {
    tcflag_t c_iflag;  // 输入模式
    tcflag_t c_oflag;  // 输出模式
    tcflag_t c_cflag;  // 控制模式
    tcflag_t c_lflag;  // 本地模式
    cc_t     c_cc[NCCS];  // 控制字符
};
```

### 3. 终端模式

#### 规范模式（Canonical Mode）
- **特点**：行缓冲，支持特殊字符
- **适用**：普通文本输入
- **标志**：`ICANON`

#### 原始模式（Raw Mode）
- **特点**：无缓冲，无特殊处理
- **适用**：游戏、全屏应用
- **标志**：`~ICANON & ~ECHO & ~ISIG`

#### 非规范模式（Non-canonical Mode）
- **特点**：可配置的最小字符数和超时
- **适用**：逐字符读取
- **标志**：`~ICANON`

### 4. 特殊字符处理

#### 控制字符映射
```c
#define VINTR    0   // Ctrl+C - 中断
#define VQUIT    1   // Ctrl+\ - 退出
#define VERASE   2   // Backspace - 擦除
#define VKILL    3   // Ctrl+U - 杀死行
#define VEOF     4   // Ctrl+D - 文件结束
#define VSUSP    10  // Ctrl+Z - 挂起
```

### 6. 常见问题

#### 问题1：终端显示混乱
**原因**：设置了错误的模式或字符。
**解决**：
```bash
# 重置终端
终端设置 sane
# 或
reset
```

#### 问题2：回显无法恢复
**原因**：脚本异常退出，未恢复设置。
**解决**：
```bash
# 手动恢复
终端设置 echo

# 或使用 trap
陷阱 '终端设置 echo' EXIT
```

#### 问题3：不是在终端中
**原因**：通过管道或重定向运行。
**解决**：
```bash
# 检查是否在终端中
如果 [ -t 0 ]; then
    终端设置 ...
否则
    显示 "错误: 需要在终端中运行" >&2
结束
```

## 📊 使用频率说明

在现代 Linux 系统中，`终端设置` (stty) 的使用频率**低到中等**。

*   **原因**：
    *   现代终端模拟器已经配置得很好。
    *   大多数用户不需要手动调整终端设置。
    *   主要在脚本编程和系统管理中使用。

*   **替代方案**：
    *   **tput**：更便携的终端控制
    *   **readline**：库级别的输入处理
    *   **ncurses**：全屏应用开发
    *   **图形界面**：GUI 应用不需要 stty

*   **推荐使用场景**：
    *   **密码输入**：禁用回显
    *   **终端修复**：重置损坏的终端
    *   **脚本编程**：特殊的终端控制需求
    *   **嵌入式系统**：串口配置
    *   **教学演示**：讲解终端工作原理

## 📜 Linux 中的定义与发展史

### 1. 电传打字机时代
stty 的概念源自电传打字机（Teletype），需要配置各种物理参数。

### 2. Unix 的终端抽象
Unix 将各种终端设备抽象为统一的接口，stty 成为配置终端的标准工具。

### 3. POSIX 标准化
POSIX 标准对 termios 结构和相关系统调用进行了规范。

### 4. 图形界面的影响
随着图形界面的普及，stty 的使用频率下降，但在服务器和嵌入式系统中仍然重要。

### 5. 现代终端模拟器
gnome-terminal、konsole 等现代终端模拟器自动处理大部分设置。

### 6. uutils 的 Rust 重写
uutils 项目用 Rust 重写了 stty，提供内存安全保证和跨平台兼容性。

### 7. 云原生时代
在容器和无服务器环境中，stty 的使用场景减少，但在调试和运维中仍有价值。

### 8. 未来的地位
虽然有更现代化的终端控制方式，但 stty 作为 POSIX 标准工具，将继续存在：
- **标准化**：POSIX 标准的一部分
- **基础性**：终端控制的基础工具
- **必要性**：某些场景下不可替代

未来，stty 可能会逐渐被更高级的库和框架取代，但在底层系统管理和脚本编程中，它仍然是重要工具。
