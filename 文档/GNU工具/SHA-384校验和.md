# SHA-384校验和 (sha384sum)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**SHA-384校验和** 在汉语编程中对应英文命令：**sha384sum**。

计算并验证文件的 SHA-384（Secure Hash Algorithm 384-bit）消息摘要。SHA-512的截断版本，提供192位安全强度，是安全性和性能的平衡选择。

**通俗理解**：
- **什么是SHA-384校验和**：就像SHA-512的"中等版本"，`SHA-384校验和` 命令会生成一个96位的十六进制字符串（384位）。它是SHA-512算法的变体，通过不同的初始值和截断输出得到。提供了比SHA-256更高的安全性，但哈希值比SHA-512短。
- **为什么需要它**：当你需要比SHA-256更高的安全性，但又不想使用完整的SHA-512时，SHA-384提供了一个很好的折中方案。它在TLS协议中被广泛使用，是安全性和效率的平衡点。
- **应用场景**：TLS/SSL证书、高安全性应用、政府标准、数字签名、需要192位安全强度的场景。

### SHA-384 特性

- **固定长度**：始终生成96位十六进制字符串（384位）
- **安全强度**：192位（高于SHA-256的128位）
- **基于SHA-512**：使用相同的算法结构
- **TLS标准**：TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384
- **NIST标准**：FIPS 180-4标准

## 🔍 底层逻辑详解

**SHA-384校验和** 的核心逻辑是 **"SHA-512的变体实现"**。

### 1. SHA-384 与 SHA-512 的关系

#### 不同的初始值
```
SHA-384 使用不同的 H0-H7 初始值
（源自素数的立方根的小数部分）
```

#### 截断输出
```
SHA-512 输出: H0 || H1 || H2 || H3 || H4 || H5 || H6 || H7  (512位)
SHA-384 输出: H0 || H1 || H2 || H3 || H4 || H5                (384位)
                                                              (省略H6, H7)
```

### 2. 安全强度对比

| 算法 | 输出长度 | 安全强度 | 碰撞阻力 | 量子抗性 |
|------|---------|---------|---------|---------|
| **SHA-256** | 256位 | 128位 | 2^128 | 64位 |
| **SHA-384** | 384位 | 192位 | 2^192 | 96位 |
| **SHA-512** | 512位 | 256位 | 2^256 | 128位 |

### 3. 为何使用 SHA-384

#### 优势
- **更高安全性**：192位强度优于SHA-256
- **TLS支持**：广泛用于TLS密码套件
- **64位优化**：在64位系统上性能优秀
- **标准化**：NIST和IETF标准

#### 劣势
- **较少普及**：不如SHA-256广泛
- **哈希值较长**：占用更多存储

### 4. TLS 中的应用

SHA-384 在 TLS 1.2 和 1.3 中被广泛使用：

```
TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384
TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
```

这些密码套件提供：
- **256位AES加密**
- **384位SHA哈希**
- **前向保密**（ECDHE）

### 5. 系统调用层面

```c
#include <openssl/sha.h>

unsigned char hash[SHA384_DIGEST_LENGTH];  // 48字节
SHA512_CTX ctx;

SHA384_Init(&ctx);
// ... 更新和最终化与SHA-512类似
SHA384_Final(hash, &ctx);
```

## 📊 使用频率说明

在现代 Linux 系统中，`SHA-384校验和` (sha384sum) 的使用频率**中等**。

*   **原因**：
    *   TLS协议广泛使用SHA-384。
    *   提供比SHA-256更高的安全性。
    *   在64位系统上性能优秀。

*   **替代方案**：
    *   **sha256sum**：更广泛支持
    *   **sha512sum**：最高安全性
    *   **b2sum**：更快的BLAKE2

*   **推荐使用场景**：
    *   **TLS/SSL**：证书和协议
    *   **高安全性应用**：需要192位强度
    *   **政府标准**：符合特定要求
    *   **64位系统**：利用性能优势
    *   **数字签名**：高安全签名

## 📜 Linux 中的定义与发展史

### 1. SHA-384 的诞生
2001年，NIST在FIPS 180-2中引入了SHA-384。

### 2. TLS 采用
- **TLS 1.2**：引入SHA-384密码套件
- **TLS 1.3**：继续使用SHA-384

### 3. 实际应用
- **HTTPS**：网站证书
- **VPN**：IPsec协议
- **电子邮件**：S/MIME
- **代码签名**：高安全签名

### 4. GNU Coreutils 的实现
GNU项目提供了sha384sum命令。

### 5. uutils 的 Rust 重写
uutils项目用Rust重写了sha384sum。

### 6. 现代应用生态
如今，`SHA-384校验和` 被应用于：
- **TLS/SSL**：网络安全
- **高安全系统**：政府、金融
- **数字证书**：PKI基础设施
- **合规性**：满足行业标准

### 7. 未来的地位
SHA-384将继续在TLS和高安全应用中发挥重要作用：
- **TLS标准**：长期支持
- **安全性**：足够应对当前威胁
- **性能**：64位系统上优秀

未来，SHA-384将与SHA-256和SHA-512一起，构成SHA-2家族的完整产品线，满足不同安全级别的需求。
