# 链接 (ln)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**链接** 在汉语编程中对应英文命令：**ln**。

给文件创建“别名”或“快捷方式”。不加参数时创建硬链接（多扇门），加 `-s` 参数创建符号链接（快捷方式）。

**通俗理解**：
- **硬链接**：就像给房间开了第二扇门，两扇门通向同一个房间。
- **符号链接**：就像在桌上放了一张纸条，写着“去某某房间”，它是一个指向目标的指引。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `链接 符号 原始数据.txt 快捷方式.lnk` |
| **英文** | `ln -s original_data.txt shortcut.lnk` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **符号** | `链接 符号 程序.sh /usr/bin/程序` | `-s` / `--symbolic` | 创建符号链接（快捷方式），而非硬链接。 |
| **强制** | `链接 符号 强制 新文件.txt 旧链接.lnk` | `-f` / `--force` | 如果目标已存在，先删除它再创建链接。 |
| **交互** | `链接 符号 交互 文件.txt 链接名` | `-i` / `--interactive` | 在覆盖已有文件前提示用户确认。 |
| **备份** | `链接 符号 备份 配置.conf 链接.conf` | `-b` / `--backup` | 在创建链接前对已存在的目标进行备份。 |
| **目录** | `链接 符号 目录 文件夹/ 链接目录/` | `-F` / `--directory` | 允许超级用户对目录创建硬链接（不推荐）。 |
| **相对路径** | `链接 符号 相对 ../文件.txt 链接` | `-r` / `--relative` | 创建相对于链接位置的符号链接。 |
| **详细输出** | `链接 符号 详细 文件.txt 链接名` | `-v` / `--verbose` | 显示每个被创建的链接名称。 |
| **帮助信息** | `链接 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `链接 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

在目标位置创建一个指向原文件的链接。如果是符号链接，它就像一个路标，告诉系统“真正的文件在那边”。

## 🔍 底层逻辑详解

**链接** 命令是 Linux 中管理文件入口的核心工具，它有两种完全不同的工作方式：

### 1. 硬链接（默认方式）
- **本质**：给同一个房间开第二扇门。
- **特点**：两扇门完全平等，没有主次之分。拆掉一扇门，房间还在。
- **限制**：不能跨楼层（跨文件系统），也不能给大厅（目录）开门。

### 2. 符号链接（加 `-s` 参数）
- **本质**：在桌上放一张写着地址的纸条（快捷方式）。
- **特点**：它是一个独立的文件，里面存的是目标的路径。如果目标被搬走了（删除或移动），纸条就失效了（变成断链）。
- **优势**：可以跨楼层（跨文件系统），也可以指向大厅（目录）。

### 3. 为什么要有两种链接？
- **硬链接**更稳固：只要还有一扇门在，数据就不会丢。适合做即时备份。
- **符号链接**更灵活：可以指向任何地方，甚至网络上的文件。适合做程序快捷方式或版本切换。
