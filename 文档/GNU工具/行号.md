# 行号 (nl)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**行号** 在汉语编程中对应英文命令：**nl**。

给文件的每一行添加行号，并输出到标准输出。

**通俗理解**：
- **自动编号**：就像是你写了一份会议记录或代码清单，为了后面讨论时方便引用（比如“请看第 15 行”），你需要给每一行标上序号。`行号` 命令就是帮你自动完成这个枯燥的编号工作。
- **智能识别**：与简单的计数器不同，`行号` 甚至可以识别哪些是正文、哪些是页脚，从而选择性地给某些行编号或不编号。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `行号 源代码.c` |
| **英文** | `nl source_code.c` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **起始编号** | `行号 --起始编号 100 文本.txt` | `-v [数字]` / `--starting-line-number=[数字]` | 设置第一行的行号，默认为 1。 |
| **递增步长** | `行号 --递增 5 文本.txt` | `-i [数字]` / `--line-increment=[数字]` | 设置行号的增加幅度，默认为 1。 |
| **编号样式** | `行号 --编号 左对齐 文本.txt` | `-n [样式]` / `--number-format=[样式]` | 设置行号的显示格式（左对齐、右对齐等）。 |
| **分隔符** | `行号 --分隔符 '::' 文本.txt` | `-s [字符串]` / `--number-separator=[字符串]` | 设置行号与正文之间的分隔符，默认为制表符。 |
| **帮助信息** | `行号 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `行号 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

输出的每一行前面都会多出一个数字。默认情况下，这个数字是右对齐的，并且与正文之间有一个制表符的距离，看起来非常整洁。

## 💻 详细用法举例

### 1. 基础用法 - 给代码加行号
```bash
行号 脚本.sh
```
**执行结果**：
```
     1	#!/bin/bash
     2	echo "Hello World"
     3	exit 0
```
**说明**：这是最常用的场景，方便在调试或审查代码时定位具体的行数。

### 2. 自定义起始行号和步长
```bash
行号 --起始编号 10 --递增 10 列表.txt
```
**执行结果**：
```
    10	第一项内容
    20	第二项内容
    30	第三项内容
```
**说明**：这在处理分段文档或需要特定编号规则的报表时非常有用。

### 3. 改变行号的显示位置
```bash
行号 --编号 左对齐 --宽度 3 简短文本.txt
```
**执行结果**：
```
1   第一行
2   第二行
3   第三行
```
**说明**：通过 `--编号` 和 `--宽度` 参数，你可以让行号紧贴着文字，节省屏幕空间。

### 4. 配合管道使用
```bash
查看 错误日志.log | 行号
```
**执行结果**：实时给输出的日志流加上行号。当你需要向同事报告“第 50 行那个报错”时，这个功能简直是神器。

## 🔍 底层逻辑详解

**行号** (nl) 的核心逻辑是 **"状态机计数"** 与 **"格式化输出"**。

### 1. 它是如何工作的？
- **逐行读取**：`行号` 从输入流中一行行读取数据。
- **计数器递增**：每读一行，它就在内部计数器上加 1（或者按你设定的步长增加）。
- **格式化拼接**：它将当前的计数值转换成字符串，根据你设定的对齐方式和宽度进行填充，最后加上分隔符和原始内容一起输出。

### 2. 高级功能：分节编号
*   **身体 (body)**：默认的编号区域。
*   **页眉 (header)** 和 **页脚 (footer)**：`行号` 允许你定义特殊的分隔符（如 `:::`）来标记页眉和页脚。你可以设置页眉和页脚不编号，而只给正文编号。这在处理包含大量元数据的复杂文档时非常专业。

### 3. 与 cat -n 的区别
虽然 `查看 -n` (cat -n) 也能加行号，但 `行号` (nl) 提供了更精细的控制：
*   **空行处理**：`nl` 可以选择是否给空行编号（`-b` 参数），而 `cat -n` 通常会跳过空行或行为不一致。
*   **样式定制**：`nl` 支持更多的对齐方式、宽度和分隔符定制，适合生成格式严格的出版级文本。

### 4. 性能表现
`行号` 是一个非常轻量级的流式处理工具。它的内存占用几乎可以忽略不计，因为它只需要维护一个整数计数器。即使处理几 GB 的文件，它也能保持极快的速度。

## 📜 Linux 中的定义与发展史

### 1. 排版与打印的需求
在计算机主要用于科学计算和文本处理的年代，程序员和作家们经常需要将代码或手稿打印出来进行审阅。为了方便指出具体的修改位置，给每一行加上编号成为了刚需。

### 2. Unix V4 的引入
`nl` 命令最早出现在 1973 年的 Unix V4 中。它的名字来源于 "Number Lines"（给行编号）。在当时，它是 `pr` (打印格式) 命令的重要补充，专门负责处理行号逻辑。

### 3. POSIX 标准的固化
POSIX 标准规定了 `nl` 必须支持分节处理（Header, Body, Footer）以及多种编号格式。这使得它在处理结构化文档时比简单的计数器更加强大。

### 4. 现代系统中的角色
在现代开发环境中，虽然 IDE 已经自带了行号显示，但 `行号` 依然在以下场景不可或缺：
*   **终端调试**：在没有图形界面的服务器上排查问题时，它是定位日志行的最佳助手。
*   **文档发布**：在生成纯文本的技术手册或法律合同时，行号能确保引用的准确性。
*   **教学演示**：老师在终端里讲解代码时，加上行号能让学员更容易跟上节奏。
