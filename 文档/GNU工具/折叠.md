# 折叠 (fold)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**折叠** 在汉语编程中对应英文命令：**fold**。

将过长的文本行在指定的宽度处强制切断（折叠），使其适应较窄的显示区域。

**通俗理解**：
- **暴力换行**：如果说 `格式化` (fmt) 是一个温柔的编辑，会找空格断行；那么 `折叠` (fold) 就像是一把无情的剪刀，不管你是不是在一个单词的中间，只要到了设定的宽度就"咔嚓"一刀切下去。
- **适应屏幕**：当你有一个超长的日志行或者一行写得很长的代码，而你的终端窗口又比较窄时，`折叠` 能强行把它折成多行，让你不用左右滚动就能看到全部内容。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `折叠 --宽度 80 长文本.txt` |
| **英文** | `fold -w 80 long_text.txt` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **指定宽度** | `折叠 --宽度 40 文本.txt` | `-w [数字]` / `--width=[数字]` | 设置每行的最大宽度，默认为 80 个字符。 |
| **按字节计算** | `折叠 --字节 --宽度 20 文本.txt` | `-b` / `--bytes` | 按字节数而不是字符数来计算宽度（处理多字节字符时有用）。 |
| **在空格处断开** | `折叠 --空格 --宽度 60 文本.txt` | `-s` / `--spaces` | 尽量在空格处断开，避免把单词切成两半。 |
| **帮助信息** | `折叠 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `折叠 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

输出内容中，任何超过指定宽度的行都会被强行切断并换到下一行。原本的一行长文本会变成多行短文本。

## 💻 详细用法举例

### 1. 基础用法 - 强制折行
```bash
折叠 --宽度 30 超长日志.log
```
**执行结果**：
```
2024-04-11 ERROR: Connectio
n failed to host 192.168.1.
100 port 8080 timeout afte
r 30 seconds.
```
**说明**：注意看，`Connection` 被切成了 `Connectio` 和 `n`。这就是 `折叠` "无情"的地方，它只管宽度，不管单词完整性。

### 2. 尝试在空格处断开（更人性化）
```bash
折叠 --空格 --宽度 30 超长日志.log
```
**执行结果**：
```
2024-04-11 ERROR:
Connection failed to host
192.168.1.100 port 8080
timeout after 30 seconds.
```
**说明**：加上 `--空格` 参数后，它会尽量寻找最近的空格进行切割，这样读起来就不会那么难受了。

### 3. 处理包含中文的文本
```bash
折叠 --字节 --宽度 20 中文文章.txt
```
**执行结果**：在处理 UTF-8 编码的中文时，使用 `--字节` 可以确保每个汉字（通常占 3 个字节）被正确对待，避免出现半个汉字的乱码情况。

### 4. 配合管道实时监控
```bash
查看 实时数据流.json | 折叠 --宽度 100
```
**执行结果**：当你在终端里查看 JSON 或 XML 这种经常没有换行的格式时，`折叠` 能让你在不使用专业解析工具的情况下，勉强看清里面的内容。

## 🔍 底层逻辑详解

**折叠** (fold) 的核心逻辑是 **"固定宽度截断"** 与 **"流式输出"**。

### 1. 它是如何工作的？
- **计数器**：`折叠` 内部维护一个字符（或字节）计数器。
- **逐字读取**：它从输入流中一个字一个字地读取。
- **触发换行**：每当计数器达到你设定的宽度（比如 80），它就输出一个换行符 `\n`，并把计数器归零，然后继续处理剩下的内容。

### 2. 三种模式的差异
*   **默认模式**：按字符数计算。在大多数现代终端下表现良好。
*   **字节模式 (-b)**：按原始 byte 数计算。这在处理二进制数据或需要精确控制存储大小时非常有用。
*   **空格模式 (-s)**：这是 `折叠` 的"妥协"模式。它会先看看当前宽度附近有没有空格，如果有，它就退回到那个空格处切一刀；如果没有，它还是会无情地切断单词。

### 3. 与 fmt 的本质区别
*   **fold (折叠)**：是"局部"的。它只关心每一行是不是太长了，它不会去合并短行，也不会识别段落。
*   **fmt (格式化)**：是"全局"的。它会重新组织整个段落的单词分布，追求整体的美观和对齐。

### 4. 为什么需要这个命令？
在现代宽屏显示器普及之前，很多老式的终端只有 80 列甚至 40 列宽。如果程序输出了一行 200 个字符的信息，在没有 `折叠` 的情况下，用户必须痛苦地左右滚动屏幕才能看完。`折叠` 解决了这个"可视性"问题。

## 📜 Linux 中的定义与发展史

### 1. 早期终端的物理限制
在 20 世纪 70 年代，计算机终端通常是基于电传打字机（Teletype）或早期的 CRT 显示器。这些设备的物理宽度非常有限，且不支持水平滚动。`fold` 应运而生，成为了让长文本适应窄屏幕的必备工具。

### 2. Unix V4 的引入
`fold` 命令最早出现在 1973 年的 Unix V4 中。它与 `fmt` 一起，构成了 Unix 早期文本处理的两大支柱：一个负责暴力折行，一个负责智能排版。

### 3. POSIX 标准的固化
POSIX 标准明确规定了 `fold` 对字节和字符的处理方式，特别是引入了 `-b` 选项来处理非 ASCII 字符集。这保证了它在不同语言环境下的稳定性。

### 4. 现代系统中的角色
虽然我们现在的显示器已经很宽了，但 `折叠` 依然在以下场景发挥重要作用：
*   **日志分析**：服务器日志往往是一行到底的巨型字符串，`折叠` 是快速预览这些日志的首选工具。
*   **嵌入式开发**：在调试串口通信或小型嵌入式设备时，屏幕往往很小，`折叠` 能帮助开发者看清输出的调试信息。
*   **数据预处理**：在进行复杂的文本挖掘前，先用 `折叠` 确保每一行都不会因为过长而导致后续的处理工具（如 `awk` 或 `grep`）崩溃。
