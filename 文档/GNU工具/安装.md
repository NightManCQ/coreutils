# 安装 (install)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**安装** 在汉语编程中对应英文命令：**install**。

复制文件并设置特定的权限、所有者等属性，常用于程序部署。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `安装 权限 755 程序.sh /usr/local/bin/` |
| **英文** | `install -m 755 program.sh /usr/local/bin/` |

## ✅ 执行结果介绍

文件被复制到目标位置且权限被设定为指定值。与普通的复制不同，安装命令通过原子性操作确保文件在创建时就具备正确的权限和属性。

## 🔍 底层逻辑详解

**安装** 命令不仅仅是简单的文件复制，它在底层执行了一系列复杂的逻辑：

### 1. 原子性创建与权限预设
- **普通复制**：先创建文件（继承默认权限），再调用 `chmod` 修改权限，存在短暂的"权限不正确"时间窗口。
- **安装命令**：先创建临时文件并设置好权限和所有者，再原子性地重命名到目标位置。用户要么看到旧文件，要么看到已具备正确权限的新文件，永远不会看到中间状态。

### 2. SELinux 安全上下文处理
- 如果系统开启了 SELinux，**安装** 会自动根据目标目录的默认安全策略，为新文件设置正确的 SELinux 标签。
- 普通复制可能会保留源文件的标签，导致新文件因标签不匹配而被系统拦截。

### 3. 符号剥离 (Stripping)
- 使用 `-s` 参数时，**安装** 会调用底层的 `strip` 工具，直接修改二进制文件内容。
- 移除调试符号表和重定位信息，减小文件体积（通常可减少30%-50%），同时防止逆向工程。

### 4. 自动目录构建
- 使用 `-D` 参数时，**安装** 会检查目标路径的每一级目录是否存在。
- 如果不存在，逐级创建并为新建目录设置特定的权限和所有者，类似于 `mkdir -p` + `cp` 的组合。

### 5. 备份机制
- 当目标文件已存在时，**安装** 可以将旧文件重命名为 `文件名~` 或带版本号的备份文件。
- 这是在确认新文件成功写入后才处理旧文件，比手动备份更安全。
