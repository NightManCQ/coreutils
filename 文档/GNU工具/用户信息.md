# 用户信息 (pinky)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**用户信息** 在汉语编程中对应英文命令：**pinky**。

显示当前登录用户的简要信息，是 `who` 命令的轻量级替代品。

**通俗理解**：
- **快速点名**：就像是你走进教室想看看谁来了，`用户信息` 命令能帮你快速扫一眼当前有哪些人登录了系统。
- **简短版 who**：它比 `who` 命令输出的内容更少、更紧凑，适合在屏幕空间有限或者你只想快速确认在线人数时使用。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `用户信息` |
| **拼音** | `yhxx` |
| **英文** | `pinky` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **长格式** | `用户信息 --长格式` | `-l` / `--long-format` | 显示更详细的用户信息（如主目录、Shell 等）。 |
| **省略头部** | `用户信息 --省略头部` | `-b` / `--omit-header` | 不显示列标题行，只输出数据。 |
| **省略主目录** | `用户信息 --长格式 --省略主目录` | `-h` / `--omit-home-directory` | 在长格式下不显示主目录路径。 |
| **帮助信息** | `用户信息 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `用户信息 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

默认情况下，它会列出每个登录用户的用户名、终端设备、登录时间以及远程主机地址（如果是远程登录）。

## 💻 详细用法举例

### 1. 基础用法 - 快速查看在线用户
```bash
用户信息
```
**执行结果**：
```
登录名     名称           I 什么            何时             哪里
root       root           *  tty1           Apr 11 09:00     
user1      User One       *  pts/0          Apr 11 10:30     192.168.1.100
```
**说明**：这是最简单的用法，一眼就能看出谁在什么时候从哪里登录了系统。

### 2. 使用长格式获取更多信息
```bash
用户信息 --长格式 user1
```
**执行结果**：
```
登录名: user1
姓名: User One
目录: /home/user1
Shell: /bin/bash
从不登录。
计划文件为空。
邮件已读于 Thu Apr 11 10:00:00 2024
```
**说明**：长格式会显示该用户的个人配置信息，包括主目录、默认 Shell 甚至邮件状态。

### 3. 配合脚本使用（省略头部）
```bash
用户信息 --省略头部 | 剪切 --字段 1
```
**执行结果**：只输出当前在线用户的用户名列表。这在编写自动化监控脚本时非常有用。

## 🔍 底层逻辑详解

**用户信息** (pinky) 的核心逻辑是 **"读取 utmp 记录"** 与 **"格式化输出"**。

### 1. 它是如何知道谁在线的？
*   **utmp 文件**：Linux 系统有一个特殊的二进制文件（通常是 `/var/run/utmp`），专门用来记录当前的登录会话。
*   **实时扫描**：`用户信息` 命令会打开这个文件，逐条读取里面的记录，过滤出那些状态为"活跃"的会话。
*   **解析细节**：它会从记录中提取出用户名、终端名、登录时间戳和远程 IP 地址。

### 2. 长格式的信息来源
当你使用 `-l` 参数时，`用户信息` 还会去读取 `/etc/passwd` 文件来获取用户的主目录和 Shell 信息，甚至还会检查用户的 `.plan` 和 `.project` 文件（如果存在的话）。

### 3. 与 who 和 w 的区别
*   **who**：输出标准的登录信息，格式比较固定。
*   **w**：输出最详细的信息，还包括用户当前正在运行的进程和系统负载。
*   **pinky (用户信息)**：介于两者之间，但更偏向于简洁。它的名字其实来源于一个老式的 Unix 笑话（"Pinky finger"是最小的手指，暗示它是最小的 who 命令）。

## 📊 使用频率说明

在现代 Linux 系统中，`用户信息` (pinky) 的使用频率**较低**。
*   **原因**：大多数管理员更习惯使用 `who` 或 `w`，因为它们的输出更直观，且功能更全面。
*   **替代方案**：推荐使用 `who` 查看登录用户，使用 `w` 查看详细信息（包括进程和负载）。
*   **特殊场景**：在极简嵌入式系统、需要兼容旧版 Unix 脚本，或者想要快速扫一眼在线人数时使用。

## 📜 Linux 中的定义与发展史

### 1. BSD 的幽默传统
`pinky` 命令最早出现在 BSD Unix 中。它的名字来源于加州大学伯克利分校的一个内部笑话：既然有 `finger` (手指) 命令来查询用户详细信息，那么比小拇指还小的 "Pinky" (小指) 自然就是最简版的查询工具了。

### 2. GNU Coreutils 的收录
GNU 项目组将 `pinky` 纳入了 Coreutils，主要是为了提供一套完整的、兼容 BSD 行为的工具集。虽然在功能上它与 `who` 高度重叠，但它保留了独特的长格式输出和对 `.plan` 文件的支持。

### 3. 现代系统中的角色
随着图形化界面和更先进的监控工具（如 `htop`、`last`）的普及，`用户信息` 在日常运维中的存在感越来越弱。但在一些极简主义的嵌入式系统或老旧的 Unix 环境中，它依然是一个轻便的在线用户查询工具。
