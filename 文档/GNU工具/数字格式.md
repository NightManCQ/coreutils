# 数字格式 (numfmt)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**数字格式** 在汉语编程中对应英文命令：**numfmt**。

重新格式化数字，支持在不同单位之间转换（如字节到人类可读格式）。用于将大数字转换为更易读的格式，或在不同单位系统之间转换。

**通俗理解**：
- **什么是数字格式**：就像一个"数字翻译器"，`数字格式` 命令可以把难以阅读的大数字转换成人类友好的格式。比如把 1073741824 字节转换成 "1.1G"，或者把 "5K" 转换成 5120。它支持多种单位系统（SI、IEC），让数字更容易理解和比较。
- **为什么需要它**：当你处理文件大小、内存用量、磁盘空间等大数字时，原始的数字（如 1073741824）很难快速理解。使用 `数字格式` 可以将其转换为 "1.1G" 这样直观的格式。反过来，你也可以将 "5M" 这样的格式转换为精确的字节数，便于计算。
- **应用场景**：文件大小显示、系统监控、日志分析、数据报告、自动化脚本、单位转换。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `数字格式 --from=iec 1073741824` |
| **拼音** | `szgs` |
| **英文** | `numfmt --from=iec 1073741824` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
|------|------|---------|---------|
| **从单位转换** | `数字格式 --from=iec 1073741824` | `--from=UNIT` | 从指定单位系统解析输入。支持：si、iec、iec-i、none。 |
| **到单位转换** | `数字格式 --to=iec 1073741824` | `--to=UNIT` | 转换到指定单位系统输出。 |
| **后缀添加** | `数字格式 --suffix=B 1073741824` | `--suffix=SUFFIX` | 添加后缀（如 B 表示字节）。 |
| **小数位数** | `数字格式 --format=%.2f 1073741824` | `--format=FORMAT` | 设置输出格式（printf风格）。 |
| **填充宽度** | `数字格式 --padding=10 1024` | `--padding=N` | 设置最小字段宽度。 |
| **分隔符** | `数字格式 --delimiter=, 数据` | `-d CHAR` / `--delimiter=CHAR` | 设置字段分隔符。 |
| **字段选择** | `数字格式 --field=2 数据` | `-f FIELDS` / `--field=FIELDS` | 选择要转换的字段。 |
| **帮助信息** | `数字格式 --帮助` | `--help` | 显示完整的帮助文档。 |
| **版本信息** | `数字格式 --版本` | `--version` | 显示版本号并退出。 |

### 单位系统

| 单位系统 | 基数 | 示例 | 说明 |
|---------|------|------|------|
| **si** | 1000 | 1K = 1000 | 国际单位制（SI） |
| **iec** | 1024 | 1K = 1024 | IEC标准（二进制） |
| **iec-i** | 1024 | 1Ki = 1024 | IEC带i后缀 |
| **none** | - | 无转换 | 不转换，仅格式化 |

### 单位后缀

| 后缀 | SI值 | IEC值 | 说明 |
|------|------|-------|------|
| **K/k** | 1000 | 1024 | 千 |
| **M** | 1,000,000 | 1,048,576 | 兆 |
| **G** | 10^9 | 2^30 | 吉 |
| **T** | 10^12 | 2^40 | 太 |
| **P** | 10^15 | 2^50 | 拍 |
| **E** | 10^18 | 2^60 | 艾 |

## 🔍 底层逻辑详解

**数字格式** 的核心逻辑是 **"数字单位转换和格式化"**。

### 1. 工作原理

当执行 `数字格式` 命令时：
1. **解析输入**：读取数字和单位后缀。
2. **识别单位**：根据 --from 参数确定单位系统。
3. **转换为基准**：将输入转换为基本单位（如字节）。
4. **转换为目标**：根据 --to 参数转换到目标单位。
5. **格式化输出**：按照指定格式输出结果。

### 2. 单位转换算法

#### SI 单位（十进制）
```
1K = 1000^1 = 1,000
1M = 1000^2 = 1,000,000
1G = 1000^3 = 1,000,000,000
```

#### IEC 单位（二进制）
```
1K = 1024^1 = 1,024
1M = 1024^2 = 1,048,576
1G = 1024^3 = 1,073,741,824
```

### 3. 转换公式

#### 从单位到字节
```c
double 解析单位(const char *输入, int 单位系统) {
    double 数值;
    char 单位;
    
    sscanf(输入, "%lf%c", &数值, &单位);
    
    switch (单位) {
        case 'K': return 数值 * (单位系统 == SI ? 1000 : 1024);
        case 'M': return 数值 * pow(基数, 2);
        case 'G': return 数值 * pow(基数, 3);
        // ...
    }
    
    return 数值;
}
```

#### 从字节到单位
```c
void 格式化单位(double 字节, int 单位系统) {
    const char *单位[] = {"", "K", "M", "G", "T", "P", "E"};
    double 基数 = (单位系统 == SI) ? 1000 : 1024;
    
    int 级别 = 0;
    double 值 = 字节;
    
    while (值 >= 基数 && 级别 < 6) {
        值 /= 基数;
        级别++;
    }
    
    printf("%.1f%s", 值, 单位[级别]);
}
```

### 4. 与相关命令的对比

| 命令 | 功能 | 优势 | 劣势 |
|------|------|------|------|
| **numfmt** | 数字格式化 | 灵活、标准化 | 学习曲线 |
| **awk** | 文本处理 | 通用 | 需手动实现转换 |
| **printf** | 格式化输出 | 内置 | 无单位转换 |
| **human-readable** | 自定义脚本 | 定制化 | 非标准 |

### 5. 常见问题

#### 问题1：SI vs IEC 混淆
**原因**：不清楚两种单位系统的区别。
**解决**：
```bash
# SI（硬盘厂商常用）
数字格式 --to=si 1000000000  # 1.0G

# IEC（操作系统常用）
数字格式 --to=iec 1073741824  # 1.1G
```

#### 问题2：精度丢失
**原因**：小数位数不足。
**解决**：
```bash
数字格式 --to=iec --format=%.3f 1234567890
```

#### 问题3：负数处理
**说明**：numfmt 支持负数。
```bash
数字格式 --to=iec -- -1073741824  # -1.1G
```

### 6. 最佳实践

#### 一致性
```bash
# 在整个系统中使用相同的单位系统
# 推荐：IEC（与大多数Linux工具一致）
数字格式 --to=iec --suffix=B
```

#### 精度控制
```bash
# 根据场景选择合适的小数位数
# 报告：1位小数
数字格式 --to=iec --format=%.1f

# 精确计算：3位小数
数字格式 --to=iec --format=%.3f
```

#### 脚本中使用
```bash
# 捕获错误
如果 ! 可读=$(数字格式 --to=iec $字节 2>/dev/null); then
    显示 "错误: 无效的数字" >&2
    退出 1
结束
```

## 📜 Linux 中的定义与发展史

### 1. 数字格式化的需求
随着计算机存储容量的增长，大数字的可读性成为问题。

### 2. GNU Coreutils 的创新
2013年，GNU coreutils 8.21 引入了 numfmt 命令。

### 3. SI vs IEC 之争
- **SI**：硬盘厂商使用（1GB = 10亿字节）
- **IEC**：操作系统使用（1GiB = 2^30字节）
- **numfmt**：支持两种标准

### 4. 广泛应用
- **系统监控**：资源使用报告
- **日志分析**：格式化日志数据
- **脚本编程**：自动化报告

### 5. uutils 的 Rust 重写
uutils项目用Rust重写了numfmt。

### 6. 现代应用生态
如今，`数字格式` 被应用于：
- **DevOps**：监控和报告
- **数据分析**：数据预处理
- **系统管理**：资源跟踪
- **自动化**：CI/CD 报告

### 7. 云原生时代
在云环境中，numfmt 用于：
- **云资源监控**：AWS、Azure、GCP用量
- **容器编排**：Kubernetes资源报告
- **成本优化**：资源使用分析

### 8. 未来的地位
numfmt 将继续作为有用的工具存在：
- **标准化**：GNU coreutils的一部分
- **实用性**：解决实际问题
- **灵活性**：支持多种格式

未来，虽然更多工具会内置人类可读格式，但 numfmt 作为通用工具，在脚本编程和自动化中仍有价值。
