# 运行上下文 (runcon)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**运行上下文** 在汉语编程中对应英文命令：**runcon**。

在指定的SELinux安全上下文中运行命令。用于强制进程在特定的安全标签下执行，实现细粒度的访问控制。

**通俗理解**：
- **什么是运行上下文**：就像给程序穿上特定的"安全制服"，`运行上下文` 命令可以让程序以指定的SELinux安全标签运行。SELinux是Linux的安全模块，通过给进程和文件打标签来控制访问权限。使用 `运行上下文`，你可以强制一个程序以特定的安全身份运行，即使启动它的用户有不同的权限。
- **为什么需要它**：在高度安全的环境中，你可能需要确保某些程序只能以特定的安全上下文运行，无论谁启动它们。比如，Web服务器应该以httpd_t上下文运行，数据库应该以mysqld_t上下文运行。这样可以限制程序的权限，即使被攻击，损害也被限制在最小范围。
- **应用场景**：SELinux策略 enforcement、安全隔离、特权分离、合规性要求、多租户环境、容器安全。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `运行上下文 -t httpd_t -- /usr/sbin/httpd` |
| **拼音** | `yxsxw` |
| **英文** | `runcon -t httpd_t -- /usr/sbin/httpd` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
|------|------|---------|---------|
| **指定上下文** | `运行上下文 system_u:system_r:httpd_t:s0 -- 命令` | `CONTEXT` | 完整的SELinux上下文（用户:角色:类型:级别）。 |
| **仅指定类型** | `运行上下文 -t httpd_t -- 命令` | `-t TYPE` | 只指定类型部分。 |
| **计算转换** | `运行上下文 -c -- 命令` | `-c` / `--compute` | 计算进程过渡后的上下文。 |
| **帮助信息** | `运行上下文 --帮助` | `--help` | 显示完整的帮助文档。 |
| **版本信息** | `运行上下文 --版本` | `--version` | 显示版本号并退出。 |

### SELinux 上下文结构

SELinux 上下文由四部分组成：

```
用户:角色:类型:级别
```

#### 组成部分

| 部分 | 示例 | 说明 |
|------|------|------|
| **用户** | system_u, unconfined_u | SELinux用户身份 |
| **角色** | system_r, object_r | 角色的访问权限 |
| **类型** | httpd_t, mysqld_t | 最重要的部分，决定访问权限 |
| **级别** | s0, s0-s0:c0.c1023 | MLS/MCS安全级别 |

### 常见上下文类型

| 类型 | 用途 | 说明 |
|------|------|------|
| **httpd_t** | Web服务器 | Apache/Nginx |
| **mysqld_t** | 数据库 | MySQL/MariaDB |
| **sshd_t** | SSH服务 | OpenSSH |
| **named_t** | DNS服务 | BIND |
| **postfix_t** | 邮件服务 | Postfix |
| **unconfined_t** | 无限制 | 默认上下文 |


**说明**：显示如果使用-t选项会使用的完整上下文。

### 失败情况
如果SELinux未启用：
```bash
运行上下文 -t httpd_t -- ls
```
**执行结果**：
```

```
**退出码**：1

如果上下文无效：
```bash

```
**执行结果**：
```
运行上下文: 无效的上下文: invalid_context
```
**退出码**：1

### 10. 合规性检查
```bash
#!/bin/bash
# 检查关键服务的SELinux上下文

declare -A 期望上下文
期望上下文[httpd]="httpd_t"
期望上下文[mysqld]="mysqld_t"
期望上下文[sshd]="sshd_t"

显示 "=== SELinux上下文合规性检查 ==="
显示 ""

错误=假

对于 服务 in "${!期望上下文[@]}"; do
    PID=$(pgrep -x $服务)
    
    如果 [ -z "$PID" ]; then
        显示 "⚠ $服务 未运行"
        continue
    fi
    
    实际上下文=$(ps -Z $PID | tail -1 | awk '{print $1}')
    期望类型=${期望上下文[$服务]}
    
    如果 [[ "$实际上下文" == *"$期望类型"* ]]; then
        显示 "✓ $服务: $实际上下文"
    else
        显示 "✗ $服务: 期望 *$期望类型*, 实际 $实际上下文" >&2
        错误=真
    fi
结束

如果 [ "$错误" = "真" ]; then
    显示 ""
    显示 "警告: 发现上下文不匹配" >&2
    退出 1
fi
```
**说明**：自动化合规性检查。

## 🔍 底层逻辑详解

**运行上下文** 的核心逻辑是 **"设置进程的SELinux上下文"**。

### 1. 工作原理

当执行 `运行上下文` 命令时：
1. **解析上下文**：验证提供的SELinux上下文是否有效。
2. **安全检查**：检查当前用户是否有权限切换到该上下文。
3. **设置上下文**：调用 setexeccon() 设置执行上下文。
4. **执行命令**：使用 execvp() 执行目标命令。
5. **继承**：新进程继承设置的上下文。

### 2. 系统调用层面

#### setexeccon()
```c
#include <selinux/selinux.h>

// 设置下一个exec()的上下文
if (setexeccon(context) != 0) {
    perror("setexeccon");
    exit(1);
}

// 执行命令
execvp(argv[0], argv);
```

#### 上下文验证
```c
#include <selinux/context.h>

context_t ctx = context_new(user_input);
if (ctx == NULL) {
    fprintf(stderr, "无效的上下文\n");
    exit(1);
}

// 验证各个组件
const char *用户 = context_user_get(ctx);
const char *角色 = context_role_get(ctx);
const char *类型 = context_type_get(ctx);
const char *级别 = context_range_get(ctx);
```

### 3. SELinux 架构

#### MAC vs DAC
- **DAC**（自主访问控制）：传统的Unix权限（rwx）
- **MAC**（强制访问控制）：SELinux的策略驱动控制

#### 策略类型
- **targeted**：只保护关键服务（默认）
- **minimum**：targeted的子集
- **mls**：多级安全（最严格）

### 4. 上下文转换规则

SELinux策略定义了允许的上下文转换：

```
# 示例规则
allow init_t httpd_t:process transition;
role system_r types httpd_t;
```

### 5. 与相关命令的对比

| 命令 | 功能 | 优势 | 劣势 |
|------|------|------|------|
| **runcon** | 指定上下文运行 | 精确控制 | 需要SELinux知识 |
| **sudo** | 提权执行 | 简单易用 | 基于用户而非上下文 |
| **newgrp** | 切换组 | 简单 | 仅限组切换 |
| **su** | 切换用户 | 通用 | 不涉及SELinux |
| **chcon** | 更改文件上下文 | 文件级别 | 不用于进程 |

### 6. 常见问题

#### 问题1：SELinux未启用
**原因**：系统未启用SELinux。
**解决**：
```bash
# 检查状态
sestatus

# 启用SELinux（需要重启）
# 编辑 /etc/selinux/config
SELINUX=enforcing
```

#### 问题2：权限不足
**原因**：当前用户无权切换到该上下文。
**解决**：
```bash
# 使用sudo
sudo 运行上下文 -t httpd_t -- 命令
```

#### 问题3： AVC拒绝
**原因**：SELinux策略阻止了操作。
**解决**：
```bash
# 查看审计日志
ausearch -m avc -ts recent

# 生成策略模块
audit2allow -a -M mypolicy
semodule -i mypolicy.pp
```

## 📊 使用频率说明

在现代 Linux 系统中，`运行上下文` (runcon) 的使用频率**低**。

*   **原因**：
    *   主要用于启用了SELinux的系统（RHEL、CentOS、Fedora）。
    *   大多数桌面Linux发行版默认不使用SELinux。
    *   需要专业的SELinux知识。

*   **替代方案**：
    *   **sudo**：简单的提权执行
    *   **capabilities**：细粒度权限控制
    *   **namespaces**：容器隔离
    *   **AppArmor**：Ubuntu的MAC系统
    *   **firejail**：沙箱工具

*   **推荐使用场景**：
    *   **企业服务器**：RHEL/CentOS环境
    *   **高安全要求**：政府、金融系统
    *   **合规性**：满足安全标准
    *   **多租户**：云环境隔离
    *   **容器安全**：Kubernetes SELinux策略

## 📜 Linux 中的定义与发展史

2000年，NSA开发了SELinux，后来贡献给开源社区。

- **2003年**：合并到Linux内核2.6
- **2005年**：RHEL 4默认启用
- **现在**：RHEL/CentOS/Fedora的标准组件

### 3. runcon 的发展
作为coreutils的一部分，runcon提供了命令行接口来利用SELinux功能。

### 4. 企业采用
- **政府**：美国国防部采用
- **金融**：银行和金融机构
- **云提供商**：AWS、Azure支持

### 5. 容器化时代
- **Docker**：支持SELinux标签
- **Kubernetes**：Pod安全上下文
- **OpenShift**：强制使用SELinux

### 6. uutils 的 Rust 重写
uutils项目用Rust重写了runcon。

### 7. 现代应用生态
如今，`运行上下文` 被应用于：
- **企业服务器**：生产环境保护
