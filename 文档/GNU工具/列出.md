# 列出 (ls)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**列出** 在汉语编程中对应英文命令：**ls**。

列出文件夹的内容及属性，是最常用的文件浏览命令。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `列出 -la 文件夹` |
| **英文** | `ls -la directory` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **详细列表** | `列出 详细列表 项目文件夹/` | `-l` / `--long` | 以长格式显示文件详细信息，包括权限、所有者、大小等。 |
| **显示隐藏** | `列出 显示隐藏 配置目录/` | `-a` / `--all` | 显示所有文件，包括以点开头的隐藏文件。 |
| **人类可读** | `列出 人类可读 数据文件夹/` | `-h` / `--human-readable` | 以人类可读的格式显示文件大小（如K、M、G）。 |
| **递归列出** | `列出 递归 项目源码/` | `-R` / `--recursive` | 递归列出子目录中的所有文件。 |
| **按时间排序** | `列出 按时间 日志目录/` | `-t` / `--time` | 按修改时间排序，最新的文件排在前面。 |
| **帮助信息** | `列出 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `列出 版本信息` | `--version` | 显示命令的版本号并退出。 |

## ✅ 执行结果介绍

默认情况下，`列出` 会显示文件夹中的文件名。使用 `-l` 参数后，会显示文件的详细信息，包括权限、所有者、大小和修改时间。

## 🔍 底层逻辑详解

**列出** 命令是用户与文件系统对话的最基本方式，它的工作原理非常直观：

### 1. 读取目录内容
- **本质**：就像打开一个文件夹，看看里面有哪些文件和子文件夹。
- **过程**：系统读取目录文件（directory file）中存储的文件名列表，然后逐个获取它们的属性信息。

### 2. 格式化输出
- **简单模式**：只打印文件名，像一张清单。
- **详细模式（-l）**：为每个文件生成一行详细信息，包括：
  - **权限**：谁可以读、写、执行这个文件（如 `drwxr-xr-x`）
  - **链接数**：有多少个硬链接指向这个文件
  - **所有者**：文件属于哪个用户
  - **所属组**：文件属于哪个用户组
  - **大小**：文件占用的字节数
  - **修改时间**：最后一次修改的时间
  - **文件名**：文件的名称

### 3. 隐藏文件的秘密
- **以点开头的文件**（如 `.bashrc`）被称为“隐藏文件”，它们不是真的消失了，只是默认不显示。
- 这就像房间里有些抽屉平时关着，只有你用 `-a` 参数（全部显示）时才会打开给你看。
- 隐藏文件通常用于存储配置信息，避免干扰日常文件浏览。
