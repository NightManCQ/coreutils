# 排列索引 (ptx)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**排列索引** 在汉语编程中对应英文命令：**ptx**。

生成文本的排列索引（Permuted Index），即把每一行的关键词提取出来并按字母顺序重新排列，方便快速查找。

**通俗理解**：
- **智能目录**：就像是一本书后面的"关键词索引"，它会把文章里的重要词汇都挑出来，然后按字母顺序排好，告诉你每个词出现在哪一行。
- **内容透视**：通过 `排列索引`，你可以一眼看出一篇文章或代码文件里到底讨论了哪些核心概念，而不需要从头读到尾。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `排列索引 手册.txt` |
| **拼音** | `plsy` |
| **英文** | `ptx manual.txt` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **忽略大小写** | `排列索引 --忽略大小写 文本.txt` | `-f` / `--ignore-case` | 在排序时不区分大小写字母。 |
| **输出格式** | `排列索引 --格式 roff 文本.txt` | `-F [格式]` / `--format=[格式]` | 设置输出格式，支持 roff 或 tex。 |

| **帮助信息** | `排列索引 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |

## ✅ 执行结果介绍

输出结果是一个按关键词排序的列表。每一行都会以某个关键词为中心进行旋转排列，使得你可以通过查找该词来定位原文的位置。

## 💻 详细用法举例

### 1. 基础用法 - 生成关键词索引
```bash
排列索引 技术文档.txt
```
**执行结果**：
```
...
                    数据库优化指南    数据库
        安装与配置教程    安装
                    网络协议分析    协议
...
```
**说明**：原本散落在各处的关键词现在被整齐地排列在了一起。

### 2. 忽略大小写排序
```bash
排列索引 --忽略大小写 混合文本.txt
```
**执行结果**：确保 "Apple" 和 "apple" 会被归类在一起，而不是分开排在两个地方。

### 3. 配合 grep 使用
```bash
查看 源代码.c | 排列索引 | 剪切 --字段 1 | 排序 | 去重
```
**执行结果**：这是一个经典的 Shell 管道组合，可以快速提取出源文件中出现过的所有唯一单词。

### 4. 自定义输出宽度
```bash
排列索引 --宽度 100 长句文本.txt
```
**执行结果**：如果原文句子很长，增加宽度可以防止关键词被切断，保持索引的可读性。

## 🔍 底层逻辑详解

**排列索引** (ptx) 的核心逻辑是 **"循环移位"** 与 **"字典序排序"**。

### 1. 它是如何工作的？
- **分词处理**：`排列索引` 首先将输入的每一行拆分成一个个单词。
- **生成变体**：对于每一行，它会为其中的每个"重要单词"生成一个变体。这个变体以该单词为开头，后面跟着原句的剩余部分，前面跟着原句的前半部分。
- **全局排序**：最后，它把所有生成的变体按字典序（字母顺序）进行全局排序。

### 2. 什么是"重要单词"？
*   **停用词过滤**：默认情况下，`ptx` 会忽略一些常见的虚词（如 "the", "a", "is" 等），因为这些词对检索意义不大。
*   **自定义词库**：你可以通过提供一个词库文件来告诉它哪些词应该被保留，哪些应该被忽略。

### 3. 与 sort 的区别
*   **sort (排序)**：只是简单地把整行文字按第一个字母排序。如果两句话开头不一样但内容相关，它们会被分得很开。
*   **ptx (排列索引)**：是"内容感知"的。它能把包含同一个关键词的所有句子都聚集在一起，无这个词在句子的哪个位置。

### 4. 性能表现
`排列索引` 在处理大型文本时内存占用较高，因为它需要为每一行的每个单词都生成一个索引条目。如果一个文件有 1000 行，每行 10 个词，它可能要在内存中处理 10000 条索引记录。

## 📊 使用频率说明

在现代 Linux 系统中，`排列索引` (ptx) 的使用频率**极低**。
*   **原因**：随着全文搜索引擎（如 `grep`、`ack`、`ripgrep`）和现代 IDE 的普及，人们已经不再需要通过手工生成的排列索引来查找内容了。
*   **替代方案**：推荐使用 `grep -n` 进行关键词搜索，或使用专业的文档工具（如 Sphinx、Doxygen）自动生成索引。
*   **特殊场景**：主要用于学术研究、语言学分析，或者为纯文本文档生成传统的印刷级关键词索引。

## 📜 Linux 中的定义与发展史

### 1. 图书馆学的数字化
`ptx` 的概念源自图书馆学中的" permuted title index"（轮排标题索引）。在计算机时代之前，图书管理员手工制作这种索引来帮助读者从不同角度查找书籍。

### 2. Unix V4 的引入
`ptx` 命令最早出现在 1973 年的 Unix V4 中。在当时，它是处理技术手册和科学文献的重要工具，能帮助研究人员快速定位论文中的特定术语。

### 3. POSIX 标准的固化
POSIX 标准规定了 `ptx` 的基本行为，包括其对 roff 和 tex 格式的支持。这使得它在早期的 Unix 排版工作流中占据了一席之地。

### 4. 现代系统中的角色
虽然在日常运维中几乎见不到它的身影，但 `排列索引` 依然存在于 Coreutils 中，作为一种经典的文本处理算法的参考实现。它提醒着我们，在搜索引擎出现之前，人类是如何通过巧妙的算法来组织海量信息的。
