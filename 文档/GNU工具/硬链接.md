# 硬链接 (link)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**硬链接** 在汉语编程中对应英文命令：**link**。

给同一个文件起一个“别名”。

**通俗理解**：
其实你平时创建的每一个文件（比如用 `echo` 或编辑器保存时），本质上都是一个“只有一个名字的硬链接”。
- **普通文件**：就像一个房间只有一扇门。
- **硬链接**：就像给这个房间又开了一扇新门。无论你从哪扇门进去，看到的都是同一个房间里的东西。如果你把其中一扇门拆了（删除文件名），房间本身并不会消失，你依然可以通过另一扇门进去。

所以，硬链接并不是什么高深的技术，它只是让一个文件拥有了多个可以访问的入口。

## 💻 用法举例

### 基础用法

| 语境 | 命令示例 |
| :--- | :--- |
| **中文** | `硬链接 原始数据.txt 备份链接.txt` |
| **英文** | `link original_data.txt backup_link.txt` |

### 参数详解

| 参数 | 示例 | 英文参数 | 作用说明 |
| :--- | :--- | :--- | :--- |
| **帮助信息** | `硬链接 帮助信息` | `--help` | 显示完整的帮助文档和所有可用参数。 |
| **版本信息** | `硬链接 版本信息` | `--version` | 显示命令的版本号并退出。 |

注意：`link` 命令非常简单，只接受两个参数（源文件和目标链接名），没有其他选项。

## ✅ 执行结果介绍

创建一个与原文件完全等同的新入口。两个文件名指向同一个 inode（物理数据），删除任一文件不会影响另一个，只有当所有链接都被删除时，数据才会真正被释放。

## 🔍 底层逻辑详解

**硬链接** 命令在文件系统层面创建了一个新的目录条目，它与原文件共享相同的数据块。

### 1. inode 共享机制
- **普通文件**：每个文件有唯一的 inode 编号，记录文件的元数据和数据块位置。
- **硬链接**：新文件名指向与原文件相同的 inode 编号，两者完全平等，没有"主从"关系。
- **链接计数**：文件系统会记录有多少个名字指向同一个 inode，只有当计数归零时，数据才会被删除。
- **删除行为**：删除任何一个文件名（包括所谓的"源文件"），只是减少链接计数。只要还有一个链接存在，数据就完好无损。

### 2. 与软链接的区别
| 特性 | 硬链接 (link) | 软链接 (ln -s) |
|------|--------------|---------------|
| **inode** | 与原文件相同 | 独立的 inode |
| **跨文件系统** | ❌ 不支持 | ✅ 支持 |
| **目录链接** | ❌ 不允许 | ✅ 允许 |
| **原文件删除** | ✅ 仍可访问 | ❌ 链接失效 |
| **文件大小** | 与原文件一致 | 仅存储路径字符串 |

### 3. 实际应用场景
- **数据备份**：创建硬链接作为即时备份，不占用额外空间。
- **版本管理**：多个版本指向同一份数据，节省存储空间。
- **防止误删**：即使原文件名被删除，通过硬链接仍可访问数据。

### 4. 安全注意事项：与彻底删除的联动
- **物理共享风险**：由于硬链接共享同一组物理数据块，对其中任何一个链接执行 **彻底删除 (shred)** 都会导致所有其他链接的内容同时被破坏（变为乱码）。
- **销毁建议**：如果你需要安全销毁一份机密文件，必须确保该文件没有其他硬链接存在，或者对所有相关的硬链接都执行 **彻底删除** 操作。
- **对比普通删除**：普通的 **删除 (rm)** 只是摘除一个名字，不会影响其他硬链接的数据完整性；而 **彻底删除** 是物理覆写，具有“牵一发而动全身”的效果。

### 4. 删除"源文件"的实际测试
```bash
# 1. 创建源文件和硬链接
echo "重要数据" > 原始数据.txt
硬链接 原始数据.txt 备份数据.txt

# 2. 查看两个文件的 inode（相同）
列出 -li 原始数据.txt 备份数据.txt
# 输出：
# 1234567 -rw-r--r-- 2 user group 13 4月 12 01:46 原始数据.txt
# 1234567 -rw-r--r-- 2 user group 13 4月 12 01:46 备份数据.txt

# 3. 删除所谓的"源文件"
删除 原始数据.txt

# 4. 验证备份文件仍然可用
查看 备份数据.txt
# 输出：重要数据

# 5. 查看链接数已变为 1
列出 -li 备份数据.txt
# 输出：1234567 -rw-r--r-- 1 user group 13 4月 12 01:46 备份数据.txt
```

**关键结论**：
- 硬链接创建后，**不存在"源文件"和"副本"的区别**
- 所有链接都是平等的，删除任何一个都不会影响其他链接
- 只有当最后一个链接被删除时，真正的数据才会被释放
