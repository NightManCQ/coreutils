# 真值 (true)

> **对联系统理念**: 以日常汉语为导向，精准表意，规范统一。

## 📋 核心定义

**真值** 在汉语编程中对应英文命令：**true**。

一个什么都不做且始终返回成功状态码（0）的命令，用于在脚本中表示"真"或"成功"的逻辑状态。

**通俗理解**：
- **什么是真值**：就像开关的"开"状态，或者数学中的"真"命题。`真值` 命令执行后总是返回成功（退出码0），不产生任何输出，不做任何事情。它是一个纯粹的逻辑工具。
- **为什么需要它**：在shell脚本中，我们经常需要明确表示"成功"或"真"的状态。例如，在条件判断、循环控制、逻辑运算中，`真值` 可以作为一个明确的成功信号。
- **应用场景**：脚本逻辑控制、无限循环、占位符、调试测试、初始化成功标志。

## 🔍 底层逻辑详解

**真值** 的核心逻辑是 **"始终返回成功状态码"**。

### 1. 什么是退出码？

在 Unix/Linux 系统中，每个进程结束时都会返回一个退出码（exit code/status code）：
- **0**：成功（success/true）
- **非0**：失败（failure/false），通常是 1-255 之间的值

`真值` 的设计就是始终返回 0，表示成功。

### 2. true 的工作原理

当执行 `真值` 命令时：
1. **解析参数**：检查是否有 `--help` 或 `--version` 参数。
2. **特殊处理**：如果有帮助或版本参数，显示相应信息并退出。
3. **返回成功**：否则，直接返回退出码 0。
4. **无输出**：不向 stdout 或 stderr 写入任何内容。

### 3. 系统调用层面

`真值` 的实现极其简单：

```c
// C 语言伪代码
int main(int argc, char *argv[]) {
    if (argc > 1 && (strcmp(argv[1], "--help") == 0 || 
                     strcmp(argv[1], "--version") == 0)) {
        // 显示帮助或版本信息
        return 0;
    }
    return 0;  // 始终返回 0
}
```

在 Rust 实现中（uutils）：
```rust
pub fn uumain(args: impl uucore::Args) -> UResult<()> {
    // 解析参数
    let matches = uu_app().try_get_matches_from(args)?;
    
    // 始终返回成功
    Ok(())
}
```

### 4. 与 shell 内置命令的关系

大多数 shell（bash、zsh、sh）都有内置的 `true` 命令：
- **内置版本**：更快，不需要创建新进程
- **外部版本**：`/bin/true` 或 `/usr/bin/true`，功能相同

当你输入 `true` 时，shell 优先使用内置版本。可以通过 `type true` 查看：
```bash
type true
# 输出：true is a shell builtin
```

使用外部版本：
```bash
/bin/true
/usr/bin/true
```

### 5. 冒号（:）作为 true 的别名

在 Bourne Shell 及其衍生版本中，冒号 `:` 是 `真值` 的内置别名：

```bash
:  # 等价于 真值
echo $?  # 输出 0
```

**优势**：
- 更短，输入更快
- 完全是内置的，没有外部命令版本
- 在旧脚本中非常常见

**示例**：
```bash
# 无限循环
while :; do
    echo "循环中"
done

# 占位符
if [ 条件 ]; then
    :  # 什么都不做
else
    执行操作
fi
```

### 6. 布尔逻辑在 Shell 中的应用

Shell 中的布尔逻辑基于退出码：

#### AND 逻辑（&&）
```bash
命令1 && 命令2
```
- 如果 命令1 成功（退出码 0），则执行 命令2
- 如果 命令1 失败（退出码非0），则不执行 命令2

#### OR 逻辑（||）
```bash
命令1 || 命令2
```
- 如果 命令1 失败（退出码非0），则执行 命令2
- 如果 命令1 成功（退出码 0），则不执行 命令2

#### NOT 逻辑（!）
```bash
! 命令
```
- 反转命令的退出码
- `! 真值` 等价于 `假值`
- `! 假值` 等价于 `真值`

### 7. 与 false 命令的对比

| 特性 | true | false |
|------|------|-------|
| **退出码** | 0 | 1 |
| **输出** | 无 | 无 |
| **用途** | 表示成功/真 | 表示失败/假 |
| **实现复杂度** | 极简 | 极简 |
| **性能** | 极快 | 极快 |
| **别名** | `:`（冒号） | 无 |

### 8. 常见误区

#### 误区1：true 会输出什么
**事实**：`真值` 不会产生任何输出。它只是返回成功状态码。

#### 误区2：true 会做什么实际工作
**事实**：`真值` 不做任何实际工作。它的唯一目的就是返回成功状态码。

#### 误区3：true 和 exit 0 一样
**事实**：
- `真值`：返回成功状态码，但脚本继续执行
- `exit 0`：立即终止脚本，返回成功状态码

### 10. 历史背景

- **简单性**：每个命令只做一件事
- **组合性**：简单的命令可以组合成复杂的功能
- **一致性**：统一的退出码约定

## 📜 Linux 中的定义与发展史

### 1. Unix 早期的布尔命令
1970 年代的 Unix V1 就包含了 `true` 和 `false` 命令。它们是最早的 Unix 命令之一，体现了 Unix 哲学的简洁性。

### 2. 退出码约定的确立
Unix 早期就确立了退出码的约定：
- 0 表示成功
- 非0 表示失败

这个约定一直沿用至今，成为所有 Unix-like 系统的标准。

### 3. 冒号别名的引入
Bourne Shell 引入了冒号 `:` 作为 `真值` 的内置别名，提供了更简洁的写法。这在早期的 shell 脚本中非常流行。

### 4. Shell 内置命令的出现
随着 Bourne Shell（sh）的发展，`true` 和 `false` 被实现为内置命令，以提高性能。这避免了为如此简单的操作创建新进程的开销。

### 5. POSIX 标准化
POSIX 标准对 `true` 和 `false` 的行为进行了规范：
- `true` 必须以零状态退出
- 不接受任何操作数
- 可选支持 `--help` 和 `--version`

### 6. GNU Coreutils 的实现
GNU 项目将 `true` 整合到 GNU Coreutils 包中，提供了：
- 外部命令版本（`/bin/true`）
- 帮助和版本信息
- 与其他 coreutils 命令的一致性

### 7. 现代 Shell 的优化
现代 shell（bash、zsh、fish）都对 `true` 进行了优化：
- 作为内置命令，执行速度极快
- 最小化的内存占用
- 与 shell 逻辑运算符的深度集成

### 8. uutils 的 Rust 重写
uutils 项目用 Rust 重写了 `true`，提供了：
- 内存安全保证
- 跨平台兼容性
- 与 GNU coreutils 的行为一致性

### 9. 现代应用生态
如今，`真值` 被应用于：
- **Shell 脚本**：逻辑控制、无限循环、占位符
- **测试框架**：Bats、ShUnit2 等
- **CI/CD 流水线**：条件执行、成功模拟
- **系统初始化**：systemd、init 脚本
- **教育领域**：教授 Unix 哲学和 shell 编程

### 10. 未来的地位
尽管 `真值` 是一个极其简单的命令，但它体现了 Unix 哲学的核心原则：简单、专注、可组合。在未来，只要 Unix-like 系统存在，`真值` 就会继续发挥作用。它可能不会进化出新功能，但这种稳定性正是其价值所在。特别是在无限循环和占位符场景中，`真值`（或其别名 `:`）仍然是不可替代的工具。
